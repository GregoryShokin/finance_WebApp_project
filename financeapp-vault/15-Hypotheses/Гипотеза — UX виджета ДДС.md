# UX виджета Потока: визуализация трёхслойной модели

#гипотеза #UX #виджет

**Дата создания:** 2026-04-18
**Актуализировано:** 2026-04-19 (после разбора [[Гипотеза — ДДС-модель потоков]] и перехода на простую трёхслойную модель)
**Статус:** Действует для трёхслойной модели [[Поток]]. Визуализация — два кольца + полоска инвестиций, три таба (Базовый / Свободный / Полный).
**Действующая спецификация модели:** [[Поток]] (`01-Metrics/`)

---

> Детальная спецификация виджета «Денежный поток» на странице Dashboard. Три таба, двойной донат, полоска инвестиций, контекстные подсказки. Название «ДДС» в файле осталось исторически, но документ описывает UX для простой трёхслойной модели. Явного разделения на CFO/CFF/CFI в интерфейсе нет — есть типы операций в декомпозиции.

> **Замкнутость Полного потока** обеспечивается балансово: Полный = Δ кэша на ликвидных счетах. В декомпозиции виджета добавлена строка-компенсатор «Прирост долга по кредиткам» — см. § 4.3.

## 1. Рассмотренные варианты

### 1.1 Вариант 1: Три мини-карточки

Каждый поток — отдельная карточка с мини-донатом.

**Плюсы:** Максимум информации на одном экране.
**Минусы:** Занимает много места, показывает пустые карточки при отсутствии кредитов/инвестиций.

**Вердикт:** Отвергнут.

### 1.2 Вариант 2: Водопад (Waterfall)

Один виджет: потоки перетекают слева направо к итогу.

**Плюсы:** Наглядная причинно-следственная связь.
**Минусы:** Далёк от текущего дизайна, требует нового паттерна.

**Вердикт:** Отвергнут (но красивая идея на будущее).

### 1.3 Вариант 3: Табы + общий донат (ПРИНЯТ)

Один донат с табами переключения режимов. Ближе всего к текущему дизайну.

**Плюсы:** Консистентность с текущим UI, компактность.
**Минусы:** Связь потоков видна только через переключение.

**Вердикт:** Принят.

## 3. Принцип заполнения доната

### 3.1 Общая шкала

```
max(Σ приход, Σ расход) = 75% круга (270°)
меньшая сумма = пропорционально меньше
```

### 3.2 Расчёт дуг

Для внешнего кольца (r=60, C=377):
```
income_arc = (income / max) × 283   // 75% of 377
```

Для внутреннего кольца (r=42, C=264):
```
outflow_arc = (outflow / max) × 198   // 75% of 264
```

### 3.3 Цветовая палитра

- Доходы: `#06b6d4` (cyan)
- Повседневные расходы: `#f43f5e` (rose)
- Кредиты: `#94a3b8` (slate)
- Инвестиции: `#8b5cf6` (violet) — только в полоске

### 3.4 Визуальные состояния

**Профицит (доход > расход):** Внешнее кольцо заметно длиннее внутреннего.

**Ноль:** Кольца примерно равны.

**Дефицит (расход > доход):** Внутреннее кольцо длиннее внешнего. Max = расход (75% круга), доход — пропорционально меньше.

## 4. Три таба

### 4.1 Таб «Базовый»

**Вопрос:** Могу ли я позволить текущую жизнь?

**Центр доната:** базовый поток (рег. доход − рег. расход)

**Внешнее кольцо:** регулярные доходы
**Внутреннее кольцо:** регулярные расходы
**Декомпозиция справа:**
- Рег. доходы: +180 000 ₽
- Рег. расходы: −80 000 ₽

**Подсказка:**
```
55% дохода остаётся после регулярных трат. Образ жизни устойчив.
```

**Бонус:** зона (🟢 здоровый / 🟡 узкий / 🔴 дефицит) и процент (показатель уровня жизни).

### 4.2 Таб «Свободный»

**Вопрос:** Сколько ежемесячно освобождается?

**Центр доната:** свободный поток (базовый − обязательные кредитные)

**Внешнее кольцо:** регулярные доходы
**Внутреннее кольцо:** рег. расходы + обязательные кредитные платежи
**Декомпозиция справа:**
- Рег. доходы: +180 000 ₽
- Рег. расходы: −80 000 ₽
- Кредиты: −45 000 ₽

**Полоска инвестиций (под донатом):**
```
Из 55к свободных → 30к вложено, 25к осталось
▓▓▓▓▓░░░░ 54%
```

**Подсказка:**
```
55 000 ₽ можно направить на цели или инвестиции. Уже вложено 30 000 ₽.
```

**Динамическая:** если полный сильно меньше свободного:
```
55 000 ₽ ежемесячная скорость. В этом месяце нерегулярные расходы −50к.
```

### 4.3 Таб «Полный»

**Вопрос:** Что реально произошло с кэшем?

**Центр доната:** Полный поток = Δ кэша на ликвидных счетах (regular + cash + deposit), балансово. См. [[Поток]], раздел «Полный поток».

**Внешнее кольцо:** все доходы + поступления кредитных средств (disbursement) + возвраты долгов + продажа активов
**Внутреннее кольцо:** все расходы (включая проценты по кредитам) + тело кредитных платежей + покупки активов

**Декомпозиция справа — с балансовым компенсатором, обеспечивает замкнутость** (решение 2026-04-19, путь Б):

```
Зарплата и другие доходы         +200 000
Повседневные расходы             −100 000   (включая КК-покупки)
Тело кредитных платежей            −3 000
Инвестиции (покупки активов)            0
Поступление кредитных средств           0
Прирост долга по кредиткам       +60 000   ← строка-компенсатор
─────────────────────────────────────────
Δ кэша                          +157 000
```

**Строка «Прирост долга по кредиткам»** считается балансово: `Σ(долг_КК_конец − долг_КК_начало)` по всем credit_card/installment_card счетам. Она объясняет разницу между «потрачено» и «Δ на счетах», не требуя парных проводок на каждую покупку.

**Полоска инвестиций:** абсолютная сумма вложений в настоящие инвестиции (облигации, акции, фонды) за период. Вклады в полоске не учитываются (они ликвидные).

**Подсказка:**
```
На счетах +157 000 ₽ за месяц. Из них 60к — рост долга по кредитке.
Реальный прирост своих — 97к.
```

**Обязательно:** строка-компенсатор должна быть явной. Без неё пользователь увидит расхождение между суммой строк и центральным числом.

## 5. Инвестиции — отдельно от доната

### 5.1 Обоснование

Покупка активов = смена формы денег (рубли → акции), не потеря. Если включать в донат как расход — возникает псевдо-дефицит при активных инвестициях.

**Пример:** Доход 200к, расходы 80к, купил акций на 100к. Если акции в донате → дефицит 20к (как будто впритык). Если акции отдельно → профицит 120к + инвестировано 100к.

### 5.2 Полоска под донатом

```
┌────────────────────────────────────────────┐
│ ● Инвестиции                      30 000 ₽  │
│ Из 55к свободных → 30к вложено, 25к осталось │
│ ▓▓▓▓▓░░░░░░ 54%                              │
└────────────────────────────────────────────┘
```

**Состояния:**
- Есть инвестиции: полоска показывается
- Нет инвестиций: полоска скрыта (не показываем пустую)

### 5.3 Что входит в «инвестиции»

- `investment_buy` (покупка акций, облигаций, фондов)
- Переводы на `deposit` счета (размещение на вклад)
- Возможно в будущем: покупка недвижимости, драгметаллов

## 7. Подсказки (notations)

### 7.1 Принципы

- 1-2 фразы, не больше
- Тон: наставник, поддерживающий
- Конкретные числа, не морализировать
- Цветовой блок соответствует настроению (emerald / cyan / slate)

### 7.2 Примеры для разных сценариев

**Базовый — здоровый:**
```
55% дохода остаётся после регулярных трат. Образ жизни устойчив.
```

**Базовый — узкий:**
```
После трат остаётся 4% дохода. Любая непредвиденная трата выведет в минус.
```

**Базовый — дефицит:**
```
Регулярные расходы превышают доход на 10 000 ₽/мес.
Это значит: ты живёшь не по средствам.
```

**Свободный — обычный:**
```
55 000 ₽ можно направить на цели или инвестиции. Уже вложено 30 000 ₽.
```

**Свободный — когда полный << свободного:**
```
55 000 ₽ — ежемесячная скорость генерации.
В этом месяце нерегулярные расходы −50 000 ₽.
```

**Полный — профицит:**
```
На счетах +25 000 ₽ за месяц. Все потоки учтены: жизнь, кредиты, вложения.
```

**Полный — дефицит:**
```
На счетах −120 000 ₽ за месяц. Крупная покупка или досрочное погашение.
```

## 8. Адаптация при разных сценариях ЦА

### 8.1 Первый шаг (120к/42% DTI)

**Базовый:** +5к, показатель 4% → жёлтая зона, «справляешься, но впритык».

**Свободный:** после кредита 50к остаётся 5к → жёлтая, «ежемесячно остаётся 5к».

**Полный:** ≈ +5к.

**Инвестиции:** полоска скрыта — нет средств на инвестиции.

### 8.2 Рост (200к/25% DTI)

**Базовый:** +55к, показатель 27.5% → зелёная, «устойчив».

**Свободный:** 55 − 30 = 25к после кредита → зелёная.

**Полный:** зависит от досрочных и инвестиций.

**Инвестиции:** полоска показывает: «Из 25к → 15к вложено, 10к осталось».

### 8.3 Прорыв (300к/20% DTI)

**Базовый:** +80к, показатель 27% → стабильно зелёная.

**Свободный:** 80 − 20 = 60к → зелёная.

**Полный:** активно инвестирует → показывает большие перемещения в CFI.

**Инвестиции:** крупные суммы, полоска заполнена на 80%+.

## 9. Поведение пустых потоков

### 9.1 Нет кредитов

- В табе «Свободный»: внутреннее кольцо = только расходы (slate-кредиты отсутствуют)
- Декомпозиция: строка «Кредиты: —» (серая)
- DTI на странице Health: показывает 0%

### 9.2 Нет инвестиций

- Полоска скрыта
- Декомпозиция: строка «Инвестиции: —» (серая)

### 9.3 Нет дохода (фрилансер, декрет)

- Внешнее кольцо: маленькая дуга или точка (если доход = 0)
- Центр: базовый = −80к (только расходы)
- Подсказка:
```
В этом месяце не было регулярного дохода.
Расходы 80к покрываются из накоплений.
```

### 9.4 Недостаточно истории (новый пользователь)

Все средние в системе считаются скользящим окном в 12 завершённых месяцев (см. [[Поток]]). Если у пользователя в системе меньше 12 месяцев данных, виджет ведёт себя следующим образом:

- **Базовый таб:** показывает факт по доступным месяцам, под числом — баннер-пометка «Данные за N месяцев из 12. Показатель уточнится к DD.MM.YYYY».
- **Показатель уровня жизни** (значение в процентах рядом с зоной): не отображается числом до накопления хотя бы 3 месяцев данных. Вместо числа — прочерк «—» и подпись «наберётся к DD.MM.YYYY».
- **Свободный и Полный табы:** работают от фактических данных месяца (им не нужно годовое окно).
- **Подсказка под виджетом при первом открытии** (1 раз после онбординга):
```
Виджет станет точнее, когда наберётся годовая история.
Импортируй данные за прошлый год — это займёт 5 минут и сразу даст полную картину.
```

**Обоснование:** показывать «усреднённое за 2 месяца» как диагноз — значит вводить в заблуждение. Пользователь примет случайный сезонный всплеск за образ жизни. Честнее сказать «ещё рано» и дать путь ускорить (импорт истории).

**Связка со школой:** в Модуле 0 (Знакомство) один из шагов онбординга — «Импортируй выписку за 12 месяцев», с объяснением, зачем.

## 10. Интеграция с дашбордом

### 10.1 Место виджета

Секция 1: Шапка дашборда (заменяет текущий `TrendDonut`).

### 10.2 Связь с другими виджетами

- **Доступные средства** (секция 2): показывает текущий баланс
- **Доступно для целей** (новый виджет или метрика): показывает сумму на 1-е число
- **Страница Goals:** использует свободный поток для калькулятора целей
- **Страница Health:** FI-score использует базовый поток

### 10.3 Pop-out режим (развёрнутый виджет)

При клике на виджет — модальное окно с расширенной информацией:
- История по месяцам (6-12 месяцев)
- Разбивка по категориям
- Школьные моменты
