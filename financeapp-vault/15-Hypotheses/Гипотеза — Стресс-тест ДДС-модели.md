## 0. Статус результатов (2026-04-19)

- **Часть 1 (сравнение моделей A vs B)** — историческая. После разбора гипотезы ДДС принято решение вернуться к простой трёхслойной модели (A), но с улучшениями (строка-компенсатор в декомпозиции Полного). Детали — в [[Гипотеза — ДДС-модель потоков]] § 0.
- **Часть 2 (стресс-тест трёх потоков)** — сохраняет ценность как проверка сценариев; выводы о честности Базового/Свободного/Полного остаются актуальны для трёхслойной модели.
- **Часть 3 (стресс-тест «Доступно для целей»)** — **неактуально**. Метрика отменена 2026-04-19. Причина: disbursement инвертирует формулу (пришёл кредит 500к → скачок на +500к как «доступное»). Решение — объяснять через связку показателей в школе, не делать единое число.

## Часть 1: Сравнение моделей A vs B

**Модель A (текущая):** Базовый / Свободные / Полный
**Модель B (ДДС):** Операционный / Финансовый / Инвестиционный

### Сводная таблица сравнения

| Критерий | Модель A | Модель B | Победитель |
|---|:---:|:---:|:---:|
| Простота (без кредитов) | ✅ Одно число | ❌ Три числа с нулями | A |
| Полнота картины | ❌ Слепые пятна | ✅ Видит всё | B |
| Actionability | ✅ Свободные | ✅ Каждый поток actionable | Паритет |
| Понятность для ЦА | ❌ Три абстрактных слоя | ✅ Жизнь / Кредиты / Вложения | B |
| Зависимость от классификации | ❌ Фундаментальная | ✅ Опциональная | B |
| Совместимость с мультиметрикой | ✅ Проверено | ✅ Совместимо | Паритет |
| Совместимость со школой | ❌ Сложнее объяснить | ✅ Метафора «три трубы» | B |
| Виджет | ✅ Компактный | ✅ Информативнее | B |
| «Каждый рубль — один раз» | ⚠️ Сложные правила | ✅ Автоматически | B |
| Graceful degradation | ❌ Базовый = Полный | ✅ Работает без настройки | B |
| Математическая замкнутость | ❌ Σ ≠ Δ cash | ✅ CFO+CFF+CFI = Δ cash | B |
| Фильтрация шума | ✅ Базовый стабилен | ❌ Операционный шумит | A |
| Показатель уровня жизни | ✅ Есть | ✅ Есть (рег. часть) | Паритет |

**Итог:** A — 2, B — 8, Паритет — 3

### Гибридное решение

Принимается **гибрид**:
- На уровне дашборда — структура ДДС (три потока)
- Внутри операционного потока — подразбивка на регулярный/нерегулярный (для показателя уровня жизни)

## Часть 2: Стресс-тест трёх потоков

**Базовые данные:** доход 180к (рег.), расходы 80к (рег.), кредит 45к/мес (тело 40к + % 5к).

### Сценарий 1: Накопление через месяцы

**Ситуация:**
Месяц 1: свободный = 55к, вложил 30к, осталось 25к на счёте.
Месяц 2: свободный = 55к, но на счёте 25к + 55к = 80к.

| Поток | Показывает | Честно? |
|---|---|---|
| Базовый +100к | «Образ жизни устойчив» | ✅ Правда |
| Свободный +55к | «Ежемесячно освобождается 55к» | ✅ Правда (скорость) |
| Полный +55к | «На счетах +55к» | ✅ Правда (за этот месяц) |

**Важный нюанс:** Свободный поток — **скорость генерации**, не **доступная сумма**. Доступная сумма = баланс счетов. Это разные метрики.

### Сценарий 2: Бонус 300к (нерегулярный доход)

| Поток | Без бонуса | С бонусом | Честно? |
|---|---|---|---|
| Базовый | +100к | **+100к** | ✅ Структурно ничего не изменилось |
| Свободный | +55к | **+55к** | ✅ Ежемесячная генерация та же |
| Полный | +55к | **+355к** | ✅ Реальный кэшфлоу |

### Сценарий 3: Мебель 200к из текущего месяца

**Ситуация:** Доход 200к, расходы 100к (рег.), мебель 50к (нерег.), кредит 45к.

| Поток | Показывает | Честно? |
|---|---|---|
| Базовый +100к | «Образ жизни устойчив» | ✅ Правда |
| Свободный +55к | «Ежемесячно освобождается 55к» | ✅ Правда — это скорость, не сумма |
| Полный +5к | «В этом месяце осталось 5к» | ✅ Правда |

**Ключ:** Свободный отвечает на вопрос про скорость. Полный — про факт месяца. Обе метрики честны, просто отвечают на разные вопросы.

### Сценарий 4: Получение потребкредита 500к

**Ситуация:** Пришёл кредит 500к на дебет, ещё не тратил, новый платёж начнётся со следующего месяца.

| Поток | Показывает | Честно? |
|---|---|---|
| Базовый +100к | «Образ жизни устойчив» | ✅ Правда |
| Свободный +55к | «Ежемесячно освобождается 55к» | ✅ Правда — платёж начнётся следующий месяц |
| Полный +555к | «На счетах +555к» | ✅ Правда с разбивкой: CFO +100, CFF +455, CFI 0 |

**Обязательно:** Полный поток **всегда** с разбивкой CFO/CFF/CFI. Без разбивки +555к выглядит как праздник, но 500к — заёмные.

### Сценарий 5: Досрочное погашение 200к

| Поток | Показывает | Честно? |
|---|---|---|
| Базовый +100к | Не реагирует | ✅ Досрочное не меняет образ жизни |
| Свободный +55к | Не реагирует | ✅ Ежемесячная генерация та же |
| Полный −145к | Показывает отток | ✅ Кэш ушёл |

### Сценарий 6: Фрилансер — месяц без дохода

| Поток | Показывает | Честно? |
|---|---|---|
| Базовый (факт) | −80к | ✅ Доход не покрыл расходы — факт |
| Базовый (среднее 12 мес) | +80к | ✅ Структурно устойчив |
| Свободный (факт) | −125к | ✅ Честно для этого месяца |
| Полный | −80к | ✅ Реальность |

**Решение:** Показывать два уровня:
- Текущий месяц — факт в центре доната
- 12-мес среднее — в показателе уровня жизни

### Сценарий 7: Продажа акций на 300к

| Поток | Показывает | Честно? |
|---|---|---|
| Базовый +100к | Не реагирует | ✅ Акции — не доход |
| Свободный +55к | Не реагирует | ✅ |
| Полный +355к | С разбивкой: CFO +100, CFF −45, CFI +300 | ✅ |

**Капитал как отдельная метрика:** показывает, что активы не изменились (кэш +300к, акции −300к).

### Итоговая сводка

| Сценарий | Базовый | Свободный | Полный |
|---|:---:|:---:|:---:|
| 1. Накопление | ✅ | ✅ (скорость) | ✅ |
| 2. Бонус | ✅ | ✅ | ✅ |
| 3. Мебель из месяца | ✅ | ✅ | ✅ |
| 4. Получение кредита | ✅ | ✅ | ✅ с разбивкой |
| 5. Досрочное | ✅ | ✅ | ✅ |
| 6. Нет дохода | ✅ факт + среднее | ✅ | ✅ |
| 7. Продажа акций | ✅ | ✅ | ✅ с разбивкой |

**Все потоки честны.** Критично: полный поток **обязательно** с разбивкой CFO/CFF/CFI.

## Часть 3: Стресс-тест метрики «Доступно для целей»

**Формула:**
```
Доступно для целей = Σ(regular + cash) − Σ(долг по кредиткам)
```
Обновляется раз в месяц, на 1-е число, по данным конца предыдущего месяца.

### Сценарий 1: Базовый

Дебет 300к, долг КК 0, вклады 500к.

```
Доступно = 300 − 0 = 300к
```

✅ Честно. Вклады не включаются — они работают на устойчивость.

### Сценарий 2: Кешбэк-пользователь

Дебет 200к, долг КК 130к (полное погашение через 5 дней).

```
Доступно = 200 − 130 = 70к
```

✅ Честно. Из 200к на дебете 130к зарезервированы под погашение КК.

### Сценарий 3: Первый шаг

Дебет 50к, долг КК 30к, вклады 0.

```
Доступно = 50 − 30 = 20к
```

✅ Технически 20к свободны. Школьный момент: «У тебя 20к свободных, но нет подушки — сначала создай резерв на вкладе».

### Сценарий 4: Богатый вкладчик

Дебет 100к, долг КК 0, вклады 3М.

```
Доступно = 100 − 0 = 100к
```

✅ Честно. Вклады не трогаем — пусть пользователь сам решит, снимать ли. На странице счетов он видит вклады и принимает решение.

### Сценарий 5: Revolving КК

Дебет 200к, долг КК 80к, пользователь платит только минимум.

```
Доступно = 200 − 80 = 120к
```

✅ Педагогически верно. Метрика говорит: «120к свободных. Ещё 80к зарезервированы под долг КК. Погаси — будет 200к.» Наставническая позиция.

### Сценарий 6: Две кредитки

КК1: 50к, грейс ещё 20 дней. КК2: 30к, грейс истёк, капают проценты.

```
Доступно = дебет − 80к
```

✅ Оба долга нужно вернуть, разница в срочности не меняет формулу.

### Сценарий 7: КК с лимитом, нулевой долг

Лимит 500к, долг 0. Дебет 100к.

```
Доступно = 100 − 0 = 100к
```

✅ Лимит не долг. Пока не потрачено — не вычитается.

### Сценарий 8: Только что получена зарплата (моментальный снимок)

Дебет 250к (вчера пришла ЗП 200к), долг КК 0.

**Без фиксации:**
```
Доступно сейчас = 250к
```
❌ Завышает — 80к уйдёт на расходы, 45к на кредит в этом месяце.

**С фиксацией раз в месяц:**
```
Доступно на 1 апреля = баланс на 31 марта = 125к (корректно)
```
✅ На конец месяца все обязательства уже списаны.

### Сценарий 9: Вклады = 0, кредитки = 0

Дебет 200к, долг КК 0, вклады 0.

```
Доступно = 200к
```

✅ Честно. Все 200к свободны для любых решений.

### Сценарий 10: Отрицательный баланс (овердрафт)

Дебет −50к (ушёл в овердрафт), долг КК 0.

```
Доступно = −50 − 0 = −50к
```

⚠️ Отрицательное число. Школьный момент: «Ты в минусе. Сначала выйди из овердрафта, потом думай о целях».

### Итоговая сводка

| # | Сценарий | Формула | Честно? |
|---|---|---|:---:|
| 1 | Базовый | 300к | ✅ |
| 2 | Кешбэк | 70к | ✅ |
| 3 | Первый шаг | 20к | ✅ + школа |
| 4 | Богатый вкладчик | 100к | ✅ |
| 5 | Revolving КК | 120к | ✅ |
| 6 | Две кредитки | Дебет − 80к | ✅ |
| 7 | КК без долга | 100к | ✅ |
| 8 | Только что ЗП | Фиксация раз в месяц | ✅ |
| 9 | Без долгов | 200к | ✅ |
| 10 | Овердрафт | −50к | ✅ + школа |

**Все 10 сценариев проходят.**

## Часть 4: Ключевые выводы стресс-теста

### 4.1 Три потока

1. **Все три потока дают честные ответы** при правильном понимании, на какой вопрос каждый отвечает.
2. **Свободный поток** — это скорость, не сумма. Подсказка должна это отражать.
3. **Полный поток** — всегда с разбивкой CFO/CFF/CFI. Без разбивки disbursement выглядит как доход.
4. **Базовый поток** — с двумя представлениями: факт месяца + 12-мес среднее.

### 4.2 Доступно для целей

1. **Вклады не включаются** — это отдельный мир, работают на устойчивость.
2. **Только кредитки вычитаются** — остальные долги обслуживаются через свободный поток.
3. **Обновление раз в месяц** — убирает проблему моментального снимка в начале месяца.
4. **Три инструмента — три вопроса:**
   - Свободный поток: «Сколько я генерирую?»
   - Полный поток: «Что произошло?»
   - Доступно для целей: «Сколько я могу направить?»

### 4.3 Математическая замкнутость

```
CFO + CFF + CFI = Δ кэша на ликвидных счетах
```

Главное преимущество ДДС-модели: ничего не прячется, все деньги учтены.

## Часть 5: Нерешённые риски

### 5.1 Срочные вклады

Если пользователь захочет «взять из вкладов» — система не знает, какие вклады срочные. Пока решено: вклады вообще не включаются в «Доступно для целей». Пользователь видит их на странице счетов и принимает решение сам.

### 5.2 Интерпретация пользователем

«Свободный поток = 55к» может быть неверно интерпретирован как «у меня есть 55к свободных». Решение: подсказка «**Ежемесячно** освобождается 55к» + рядом «Доступно для целей: 120к».

### 5.3 Кредитки с очень большим долгом

Если долг по кредитке больше баланса на дебете — «Доступно для целей» становится отрицательным. Это честный сигнал, но эмоционально тяжёлый. Школьный момент должен смягчить.
