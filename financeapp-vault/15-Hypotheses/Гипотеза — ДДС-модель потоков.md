### Почему отклонено

Стресс-тест гипотезы дал 8 претензий (5 основных + 3 дополнительных). Все они были решаемы — но после внесения всех правок ДДС семантически **свелась к простой трёхслойной модели**, при этом сохранила техническую сложность:

- **CFO** после решения «проценты = regular expense» ≈ Базовый поток (с нерегулярными)
- **CFF** свёлся к «тело кредитов + disbursement + личные долги» — это вычитаемое для Свободных
- **CFI** свёлся к настоящим инвестициям (облигации, акции, фонды), без вкладов
- **Замкнутость** `CFO+CFF+CFI = Δ кэша` потребовала парных `credit_disbursement` на каждую покупку с кредитной/рассрочной карты, balance differencing в нормализаторе, миграции исторических данных

Простая трёхслойная модель обеспечивает то же самое дешевле:
- Полный поток = Δ кэша **по построению** (балансовый расчёт)
- Декомпозиция в виджете замыкается одной балансовой строкой «Прирост долга по кредиткам» (компенсатор, путь Б)
- Не требует парных проводок, не требует миграции истории

## 1. Суть гипотезы

Текущая трёхслойная модель (Базовый / Свободные / Полный) **остаётся**. К ней **добавляется** структура ДДС — финансовый и инвестиционный потоки, которые раньше были невидимы («кредит = отдельный мир»).

### Что меняется
- Финансовые операции (получение кредита, платежи, досрочные, личные долги) становятся **видимыми** как отдельный поток
- Инвестиционные операции (покупка/продажа активов, вклады) становятся **видимыми** как отдельный поток
- Сумма трёх потоков = Δ кэша на счетах (математическая замкнутость)

### Что не меняется
- Операционный поток (расчёт базового и свободного потоков внутри него) работает как сейчас
- Флаг регулярности `Category.regularity` сохраняется
- Мультиметрика (DTI, Капитал, Запас, FI-score) не меняется

## 2. Три типа потоков (ДДС)

### 2.1 Операционный поток (CFO)

**Вопрос:** Что приносит и забирает повседневная жизнь?

```
Операционный = Σ доходов − Σ расходов (кроме технических операций)
```

**Что входит:**
- Все `income` (зарплата, фриланс, бонусы, подарки) — приход
- Все `expense` (продукты, аренда, кафе, мебель, ремонт) — расход
- **Проценты по кредитам** — регулярный `expense` в системной категории «Проценты по кредитам» (решение 2026-04-19, см. § 11.1)
- Нерегулярные расходы попадают сюда же (мебель, техника, крупные покупки)

**Что НЕ входит:**
- Тело кредитов → финансовый поток (через `transfer`)
- Покупки с кредитной/рассрочной карты: `expense` идёт в CFO, парный `credit_disbursement` на величину использованного лимита генерируется в CFF — обеспечивает замкнутость CFO+CFF+CFI = Δ кэша (решение 2026-04-19)

**Внутренняя декомпозиция:**
```
Операционный:
  Регулярный   → базовый поток → показатель уровня жизни
  Нерегулярный → часть полного потока
```

### 2.2 Финансовый поток (CFF)

**Вопрос:** Сколько денег прошло через долговые операции?

```
Финансовый = Σ disbursement (включая парные при покупках с кредитных карт)
             − Σ тела кредитов
             − Σ досрочных − Σ личных долгов выданных
             + Σ возвратов личных долгов
```

**Что входит:**
- `credit_disbursement` (получение кредита на дебет) — приход
- Парные `credit_disbursement` при покупках с кредитных/рассрочных карт на величину использованного лимита — приход (см. § 2.1)
- Тела платежей по кредитам — расход (transfer)
- `credit_early_repayment` (досрочное погашение) — расход
- `debt_give` (дал в долг) — расход
- `debt_take` (взял в долг) — приход
- `debt_return` (вернул долг) — расход
- `debt_collect` (получил возврат долга) — приход

> Проценты по кредитам — **в операционный поток (CFO)**, как регулярный `expense`. Решение от 2026-04-19, см. § 11.1.

**Характер потока:**
- Шумный, зависит от разовых событий
- Может быть +500к в месяц получения кредита и −5к в обычный месяц
- Не подходит для оценки нагрузки — для этого есть DTI

### 2.3 Инвестиционный поток (CFI)

**Вопрос:** Сколько я вкладываю в будущее?

```
Инвестиционный = Σ продаж активов − Σ покупок активов
```

**Что входит:**
- `investment_buy` (покупка облигаций, акций, фондов) — расход
- `investment_sell` (продажа активов) — приход
- Покупка недвижимости, драгметаллов, дорогостоящих активов (если учёт ведётся) — расход

**Что НЕ входит:**
- **Вклады** (`deposit` счета) — это ликвидные средства, не инвестиции. Переводы на вклад и с вклада = `transfer` между ликвидными счетами, не попадают в поток. См. блок «Вклады — не инвестиции» ниже.

**Важно:** Настоящие инвестиции — **не расход** в обычном смысле. Деньги не теряются, они меняют форму (рубли → акции). Поэтому визуально они показываются **отдельно**, не создавая дефицит на донате.

> [!important] Вклады — не инвестиции (решение 2026-04-19)
> На российском рынке реальная доходность по вкладам с учётом инфляции отрицательная. Вклад — это инструмент **ликвидности и защиты от обесценивания**, часть подушки безопасности. Настоящая инвестиция начинается с облигаций.
>
> Это принципиально для педагогики школы: виджет Потока не должен называть пополнение вклада «инвестицией» — иначе пользователь смешивает разные этажи [[Пирамида финансовых приоритетов]] (подушка — нижний этаж, инвестиции — верхний).
>
> **Следствие для модели:**
> - `deposit` счёт считается ликвидным (попадает в состав regular+cash при расчёте Δ кэша)
> - Переводы на/с вклада = `transfer`, НЕ в CFI
> - Полоска «Инвестиции» под донатом (§ 5 UX-гипотезы) показывает только реальные инвестиции

### 2.4 Математическая замкнутость

```
CFO + CFF + CFI = Δ кэша на ликвидных счетах за период
```

Это главное преимущество модели ДДС: ничего не прячется, все деньги учтены. В текущей модели полный поток ≠ Δ кэша при наличии кредитных операций.

## 3. Три режима виджета (табы)

Один донат, три таба. Каждый таб отвечает на свой вопрос, меняет центр доната и декомпозицию.

### 3.1 Таб «Базовый»

**Вопрос:** Могу ли я позволить текущую жизнь?

```
Базовый = регулярные доходы − регулярные расходы
```

**Донат:**
- Внешнее кольцо: регулярные доходы
- Внутреннее кольцо: регулярные расходы
- Центр: базовый поток

**Подсказка:** «55% дохода остаётся после регулярных трат. Образ жизни устойчив.»

**Дополнительно:** Показатель уровня жизни (зона + процент).

### 3.2 Таб «Свободный»

**Вопрос:** Сколько ежемесячно освобождается?

```
Свободный = Базовый − тело обязательных кредитных платежей
```

> Проценты уже учтены внутри Базового (как регулярный `expense`). Вычитать их повторно нельзя — возникнет двойной учёт. Решение 2026-04-19, см. § 11.1.

**Донат:**
- Внешнее кольцо: регулярные доходы
- Внутреннее кольцо: регулярные расходы (включая проценты) + тело обязательных кредитных
- Центр: свободный поток

**Подсказка:** «55 000 ₽ можно направить на цели или инвестиции. Уже вложено 30 000 ₽.»

**Важно:** Свободный поток — это **скорость генерации**, не доступная сумма. Если нерегулярные расходы в текущем месяце высоки, добавляется уточнение: «В этом месяце нерегулярные расходы −50 000 ₽».

**Полоска инвестиций:** показывает, сколько из свободных уже вложено за период.

### 3.3 Таб «Полный»

**Вопрос:** Что реально произошло с кэшем?

```
Полный = CFO + CFF + CFI = Δ кэша
```

**Донат:**
- Внешнее кольцо: все доходы (+ disbursement, возвраты долгов, продажа активов)
- Внутреннее кольцо: все расходы (включая проценты) + тело кредитов
- Центр: Δ кэша

**Подсказка:** «На счетах +25 000 ₽ за месяц. Все потоки учтены: жизнь, кредиты, вложения.»

**Полоска инвестиций:** абсолютная сумма вложений за период.

**Важно:** Полный поток **всегда** с разбивкой по ДДС (CFO / CFF / CFI). Без разбивки disbursement и продажа активов выглядят как доход, что вводит в заблуждение.

## 4. Визуализация (вариант Б — принят)

### 4.1 Принцип заполнения колец

```
max(Σ приход, Σ расход) = 75% круга
меньшее заполняется пропорционально
```

- Внешнее кольцо длиннее → профицит
- Внутреннее кольцо длиннее → дефицит
- Кольца равны → на нуле

### 4.2 Структура виджета

```
┌──────────────────────────────────────────────────┐
│  Денежный поток                                   │
│  ┌──────────────────┐                             │
│  │ [Базовый][Свободный][Полный]  (табы)           │
│  └──────────────────┘                             │
│                                                    │
│   ┌───────┐      Доходы         +120 000          │
│   │ донат │ ────────────                           │
│   │       │      Расходы          −65 000          │
│   │ +55к  │      Тело кредитов    −50 000          │
│   └───────┘      Инвестиции       —                │
│                                                    │
│  ▓▓▓▓▓░░░░░ Инвестиции: 30 000 (54%)              │
│                                                    │
│  ┌────────────────────────────────────────────┐   │
│  │ Подсказка (1-2 фразы под текущий таб)      │   │
│  └────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────┘
```

### 4.3 Цветовая схема

- Доходы: `#06b6d4` (cyan)
- Повседневные расходы: `#f43f5e` (rose)
- Кредиты: `#94a3b8` (slate)
- Инвестиции: `#8b5cf6` (violet) — только в полоске, не на донате

### 4.4 Инвестиции отдельно

Инвестиции **не показываются** во внутреннем кольце доната — иначе покупка активов на 100к выглядит как дефицит. Вместо этого — отдельная полоска под донатом:

```
Из 100к свободных → 80к вложено, 20к осталось
▓▓▓▓▓▓▓▓░░ 80%
```

При отсутствии инвестиций полоска скрыта.

## 5. Соотношение потоков и метрик

### 5.1 Архитектурная схема

```
┌─────────────────────────────────────────────────────┐
│  ПОТОКИ (ДДС)              МЕТРИКИ (аналитика)       │
│                                                       │
│  Операционный ──рег.часть──→ Показатель ур. жизни    │
│       │                                               │
│       └──рег.часть──┐                                 │
│                      ├──→ Свободный поток (скорость)  │
│  Финансовый ─────────┘                                │
│       │          обяз.часть                           │
│       └──────────────────→ DTI                        │
│                                                       │
│  Инвестиционный ────────→  (будущее: ROI, структура)  │
│                                                       │
│  Все три потока ───────→  Капитал, Запас, FI-score    │
│                                                       │
│  Баланс счетов ────────→  Доступно для целей          │
│                            (раз в месяц)              │
└─────────────────────────────────────────────────────┘
```

### 5.2 Зачем разделение «потоки vs метрики»

- **Потоки** = данные, первый слой. Показывают, **что происходит** с деньгами.
- **Метрики** = интерпретация, второй слой. Показывают, **хорошо ли это**.

Один и тот же поток может быть источником нескольких метрик. Например, операционный поток → базовый поток → показатель уровня жизни → норма сбережений → компонент FI-score.

### 5.3 Отношение к DTI

Финансовый поток и DTI — **разные метрики**:

| | Финансовый поток | DTI |
|---|---|---|
| Включает | Всё: disbursement, платежи, досрочные, личные долги | Только обязательные платежи |
| Стабильность | Скачет | Стабилен |
| Вопрос | «Сколько денег прошло через долги?» | «Какая доля дохода заблокирована?» |
| Actionability | Низкая | Высокая (зоны) |

DTI остаётся отдельной карточкой на дашборде.

## 6. ~~«Доступно для целей»~~ — отменено 2026-04-19

**Решение:** отдельная метрика «Доступно для целей» как единое число **не делается**.

**Причина.** Любая формула такого вида ломается на кредитных операциях. Пример: пришёл кредит 500к на дебет → баланс ликвидных вырос на 500к → метрика показывает «доступно +500к на цели», пользователь чувствует себя королём, а по факту он просто взял взаймы. Никакая одна формула не умеет отделять «мои деньги» от «деньги, которые я должен вернуть», потому что технически они на одном счёте.

**Что делается вместо этого.** Пользователю объясняется картина через **связку нескольких показателей**, каждый отвечает на свой вопрос:

| Вопрос | Инструмент |
|---|---|
| Сколько я генерирую ежемесячно? | Свободный поток (скорость) |
| Что реально произошло с кэшем в этом месяце? | Полный поток (с разбивкой CFO/CFF/CFI) |
| Какая доля дохода заблокирована долгами? | DTI |
| Сколько я продержусь без дохода? | Запас |
| Богатею или беднею? | Капитал |

Переосмысление смысла «сколько я могу потратить» происходит в **рамках школы** — модули 1 («Денежный поток») и 3 («Долги») объясняют, как не путать кредитные средства со своими.

## 7. Маппинг operation_type → потоки

| operation_type | Поток | Комментарий |
|---|---|---|
| `income` (regular) | CFO (+) | Зарплата, фриланс — основная деятельность |
| `income` (irregular) | CFO (+) | Бонусы, подарки |
| `expense` (regular) | CFO (−) | Продукты, аренда, подписки |
| `expense` (irregular) | CFO (−) | Мебель, ремонт, подарки |
| `expense` (проценты по кредитам, системная категория) | CFO (−) | Регулярный операционный расход — плата за пользование заёмными. Решение 2026-04-19, см. § 11.1 |
| `transfer` (тело кредита) | CFF (−) | Возврат заёмных средств |
| `credit_disbursement` | CFF (+) | Привлечение заёмных средств |
| `credit_early_repayment` | CFF (−) | Досрочный возврат заёмных |
| `debt_give` | CFF (−) | Выдача личного долга |
| `debt_take` | CFF (+) | Получение личного долга |
| `debt_return` | CFF (−) | Возврат личного долга |
| `debt_collect` | CFF (+) | Получение возврата долга |
| `investment_buy` | CFI (−) | Покупка актива |
| `investment_sell` | CFI (+) | Продажа актива |
| `transfer` (на вклад) | Не в потоке | Вклад = ликвидное средство, не инвестиция. Внутреннее перемещение (решение 2026-04-19, см. § 2.3) |
| `transfer` (с вклада) | Не в потоке | То же — внутреннее перемещение |
| `transfer` (между дебетовыми) | Не в потоке | Внутреннее перемещение |

## 8. Регулярность — сохраняется

### 8.1 Почему не убираем

Рассматривали отказ от флага регулярности в пользу сглаживания за 12 месяцев. Стресс-тест показал: **это не работает** для крупных разовых событий (ремонт в кредит за 500к тратится 3 месяца, съедает средний на целый год).

### 8.2 Что остаётся

- `Category.regularity` (regular / irregular / project) — на категории
- `Transaction.is_regular` — snapshot на транзакции (нерретроактивность)
- Показатель уровня жизни считается по регулярной части операционного потока
- Свободный поток — по регулярной части операционного минус обязательные кредитные

### 8.3 Возможные упрощения (в бэклог)

Рассматривались три варианта упрощения:
1. Флаг на транзакции (не на категории)
2. Автовыбросы (система сама определяет выбросы по истории)
3. Регулярность с двумя значениями (повседневная / разовая)

Решение не принято — отложено. Оставляем текущие три значения (regular / irregular / project).

## 9. Подсказки в виджете

Каждый таб показывает короткую подсказку (1-2 фразы), объясняющую число.

### 9.1 Базовый
```
55% дохода остаётся после регулярных трат. Образ жизни устойчив.
```
Зона (зелёный / жёлтый / красный) + пояснение.

### 9.2 Свободный
```
55 000 ₽ можно направить на цели или инвестиции. Уже вложено 30 000 ₽.
```
Динамическая: если полный сильно меньше свободного → «В этом месяце нерегулярные расходы −50 000 ₽».

### 9.3 Полный
```
На счетах +25 000 ₽ за месяц. Все потоки учтены: жизнь, кредиты, вложения.
```

**Тон:** наставник, конкретные числа, без морализации. Соответствует принципам «За каждой цифрой — эмоция» и «Приложение растёт с пользователем».

## 10. Стресс-тест (результаты)

### 10.1 Три потока — честность

Прогнан по 7 сценариям: здоровая семья, Первый шаг, кешбэк-пользователь, рассрочка, потребкредит, ипотека, досрочное погашение. **Все три потока дают честные ответы.**

Ключевые нюансы:
1. Свободный поток — **скорость**, не сумма. Подсказка должна отражать это.
2. Полный поток **обязательно** с разбивкой CFO/CFF/CFI — иначе disbursement выглядит как доход.
3. Текущий месяц vs среднее — базовый показывает факт месяца, показатель уровня жизни использует 12-мес окно.

### ~~10.2 Доступно для целей — честность~~ (устарело 2026-04-19)

Результаты этого стресс-теста больше не актуальны: метрика «Доступно для целей» отменена (§ 6). При критическом разборе найден сценарий, который ломает формулу, — поступление кредита (disbursement) инвертирует метрику: пользователь видит скачок «доступного» на сумму займа и воспринимает это как свои деньги. Одной формулой ситуацию не закрыть. Решение — объяснять через связку нескольких показателей в рамках школы (см. § 6).

## 11. Открытые вопросы

### ~~11.1 Проценты по кредитам — финансовый или операционный?~~ (закрыт 2026-04-19)

**Решение: операционный (CFO).**

Проценты по кредитам — это регулярный операционный расход, системная категория «Проценты по кредитам» (`regular=true`), создаётся автоматически без запроса пользователя.

**Обоснование:**
- Проценты — плата за пользование заёмными, по факту часть стоимости жизни с кредитами
- Остаются внутри регулярных расходов Базового потока
- Из Свободного потока вычитается **только тело** обязательных кредитных — повторное вычитание процентов даёт двойной учёт
- В Полном табе визуально группируются с общими расходами; тело кредитов — отдельной полоской «Кредиты»

**Проверка на семье «Рост»** (200к / 100к рег. расходов / ипотека 45к = тело 3 + % 42):
- Базовый = 200 − (100 + 42) = +58к
- Свободный = 58 − 3 = +55к ✓
- CFO = 200 − 142 = +58; CFF = −3; CFI = 0; Σ = +55 = Δ кэша ✓

**Синхронизировано в:** § 2.2, § 3.2, § 3.3, § 7 (маппинг).

### 11.2 Названия потоков в UI

- «Жизнь / Долги / Активы»
- «Операционный / Финансовый / Инвестиционный» (корпоративный)

Не решено — требует UX-тестирования.

### 11.3 Упрощение регулярности

### 11.4 Личные долги — отдельная категория?

Сейчас в финансовом потоке. Возможно, стоит выделить в отдельный подраздел или дать пользователю выбор (личные долги — это «социальные», а не «финансовые» операции).
