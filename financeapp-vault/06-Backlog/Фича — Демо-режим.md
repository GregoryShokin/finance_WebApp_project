# Фича — Демо-режим
#бэклог #высокий-приоритет #онбординг
> Кнопка «Посмотреть на примере» на онбординге создаёт юзера с 3 месяцами реалистичных тестовых транзакций.
---
## Проблема (2026-05-03)
Новый юзер не видит ценности, пока не загрузит свою первую выписку. До этого момента:
- Dashboard пуст.
- Метрики не показательны.
- Страница импорта — единственная функциональная, но требует выписки прямо сейчас.

Это отсекает «просто посмотреть, подходит ли мне» аудиторию — они уходят за 30 секунд.

## Планируемое
### Backend
- Эндпоинт `POST /demo/seed-user` (только в dev/staging или с rate-limit 1 раз/сутки на IP в проде):
  - Создаёт нового юзера с email вида `demo+{uuid}@financeapp.local`
  - Сидит 3 месяца тестовых транзакций по одному из 3 профилей: «Первый шаг» / «Рост» / «Прорыв» (выбор юзером)
  - Реалистичные числа: зарплата, продукты, кредитные платежи, нерегулярные траты, переводы
  - Возвращает токены, юзер сразу попадает на dashboard
- Скрипт-фабрика `scripts/generate_demo_data.py` с шаблонами для каждого профиля
- Демо-юзеры помечаются `is_demo=True` (можно чистить через cron старше N дней)

### Frontend
- На странице `/login` или `/register` кнопка «Посмотреть на примере» + выбор профиля
- Маркер «Демо-режим» в шапке для демо-юзера + кнопка «Зарегистрироваться по-настоящему» (миграция данных опциональна, обычно выкидываем демо при реальной регистрации)

## Критичность
**Высокий приоритет** — нужен до публичного маркетинга.
