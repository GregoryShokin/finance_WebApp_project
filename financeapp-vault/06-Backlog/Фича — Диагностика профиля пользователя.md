# Фича — Диагностика профиля пользователя
#бэклог #высокий-приоритет #онбординг #школа
> После первого импорта система определяет аудиторный профиль («Первый шаг» / «Рост» / «Прорыв») и закрепляет его за юзером для школы и моментов.

## Проблема (2026-05-03)
Открытый вопрос в `14-Specifications/Спецификация — Структура модулей школы.md` — «как определить, с какого модуля стартовать?». Без этого:
- Юзер из «Первого шага» (DTI 42%, минус по Свободному) увидит уроки про инвестиции — раздражение.
- Юзер из «Прорыва» начнёт с «как не паниковать в долгах» — ощущение «не для меня».

Закрытие этого вопроса — обязательное условие запуска школы.

## Планируемое
### Алгоритм диагностики
После первого импорта (когда уже есть >=1 месяц данных) посчитать:
- DTI (если есть кредиты)
- Базовый поток (regular income − regular expenses)
- Свободный поток (Базовый − тело кредитов)
- Буфер устойчивости (deposit / avg expenses)

Правила определения:
| Условие | Профиль |
|---|---|
| DTI > 40% ИЛИ Свободный < 0 ИЛИ нет буфера и нет роста капитала | Первый шаг |
| DTI < 30% И Свободный > 0 И буфер 0-3 мес | Рост |
| DTI < 25% И буфер ≥ 6 мес И Свободный > 50k И уже инвестирует | Прорыв |
| Иначе | Рост (default) |

### Backend
- Поле `audience_profile` на `User` (enum: first_step / growth / breakthrough / undefined)
- Поле `profile_diagnosed_at` (timestamp)
- Сервис `AudienceProfileService.diagnose(user_id)` — вызывается после первого commit_import и при ручном запросе
- Возможность переопределить вручную в настройках (с предупреждением «контент будет адаптирован»)

### Frontend
- После первого импорта — экран «Мы определили твой профиль: Рост. Это значит, что..." с подтверждением / переопределением
- В настройках — раздел «Профиль» с возможностью переключить
- В школе — выбор стартового модуля по профилю

## Критичность
