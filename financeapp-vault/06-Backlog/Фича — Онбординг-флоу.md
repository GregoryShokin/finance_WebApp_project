# Фича — Онбординг-флоу
#бэклог #высокий-приоритет #онбординг
> Пятишаговый флоу первого знакомства: добро пожаловать → импорт или демо → категории → quick tour → ссылка на школу.
---
## Проблема (2026-05-03)
После регистрации юзер попадает сразу на пустой dashboard без объяснения, что и в каком порядке делать. Дефолтные категории создаются автоматически, но юзер о них не знает; страница импорта не объясняет, где взять выписку.

## Планируемое
### Шаги онбординга
1. **Добро пожаловать** — короткий питч (1-2 экрана): «не трекер, наставник; помогает с DTI, потоком, целями»
2. **Первая выписка ИЛИ демо** — развилка:
   - «Загрузить свою выписку» → инструкция «где взять выписку» по выбранному банку из whitelist
   - «Посмотреть на примере» → демо-режим (см. [[Фича — Демо-режим]])
3. **Дефолтные категории** — показать созданные системой категории с возможностью отредактировать / убрать ненужные / добавить свои
4. **Quick tour дашборда** — 3-4 поинта: где живёт Поток, где DTI, где капитал, как открыть детали
5. **Ссылка на школу** — «начать с модуля "Первый шаг"» (после Phase 8)

### Backend
- Поле `onboarding_state` на `User` (JSON: `{step, completed, skipped_steps[]}`)
- Эндпоинты `GET/PATCH /users/me/onboarding` для трекинга прогресса
- После первого импорта — автодиагностика профиля (см. [[Фича — Диагностика профиля пользователя]])

### Frontend
- Компонент `<OnboardingWizard />` overlay с прогресс-баром
- Кнопка «Пропустить» на каждом шаге (но мягко напоминать в шапке, что онбординг не пройден)
- При повторном заходе — продолжить с прерванного шага

### UX-принципы
- Не больше 5 шагов
- Каждый шаг занимает <30 секунд
- Можно пропустить любой шаг и пройти позже из настроек
- После завершения — confetti + первый школьный момент

## Критичность
**Высокий приоритет** — без этого drop-off на первом экране очень высок.
