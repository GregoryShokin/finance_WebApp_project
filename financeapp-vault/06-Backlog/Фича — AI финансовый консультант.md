# Фича — AI финансовый консультант

#бэклог #школа #llm #поддержка

> Встроенный чат с LLM, обученной под нашу систему. Точечно объясняет пользователю его ситуацию, метрики, виджеты, решения по данным в приложении.
> **Статус:** идея, приоритет не определён.
> **Дата:** 2026-04-22

---

## Суть идеи

В приложении — чат с AI-ассистентом, который знает:
- Философию и принципы проекта (пирамида, снизу вверх, не трекер а наставник)
- Мультиметрическую систему (Поток/DTI/Капитал/Буфер, формулы, зоны)
- Кредитную логику (покупка = расход, тело = перевод, проценты = expense)
- Профили ЦА и модули школы
- Текущие данные пользователя (метрики, счета, транзакции, цели)

И умеет:
- Объяснять незнакомые термины и виджеты на данных пользователя
- Отвечать на частные вопросы («почему мой Полный в минусе, а Базовый в плюсе?»)
- Давать персональные рекомендации в рамках философии модуля пользователя
- Переводить пользователя в нужный урок школы / вебинар / раздел приложения

---

## Зачем это нам

1. **Снижает когнитивную нагрузку.** Особенно для «Первого шага» — много красных чисел и сложных концепций. AI отвечает «что это значит для тебя лично» без лекций.
2. **Масштабирует наставничество.** Живые консультации = дорого, вебинары = не точечно. AI даёт точечные ответы 24/7.
3. **Удерживает пользователя.** Вопрос → ответ прямо в приложении, не уход в Google/YouTube (где дадут противоречивые советы).
4. **Собирает данные о проблемах.** Анализ запросов → приоритет следующих уроков, школьных моментов, фич.

---

## Принципы

**Никогда не противоречит философии проекта.**
- Не советует трейдинг, крипту, МЛМ, микрозаймы
- Не рекомендует строить буфер при DTI > 60% (это противоречит «снизу вверх»)
- Не советует инвестиции пользователям «Первого шага»

**Прозрачно говорит о границах.**
- «Я не банковский консультант, проверь условия у своего банка»
- «Я не знаю твоих доходов за 3+ лет, только то что в приложении»
- Делает отсылки на конкретные уроки и вебинары

**Персонализирован под модуль.**
- Для «Первого шага» — прямой поддерживающий тон, фокус на выход из ямы
- Для «Роста» — стратегические развилки, цифры
- Для «Прорыва» — сложные концепции, критика популярных стратегий

**Работает с данными пользователя.**
Не отвечает абстрактно: «вот у тебя Базовый +5к и Полный −12к — разница в 17к, это твои нерегулярные покупки, давай посмотрим на что».

## Технические вопросы (предварительно)

1. **Какая модель?**
   - Haiku/Sonnet от Anthropic (быстро, недорого, отвечает на русском)
   - GPT-4o mini
   - Self-hosted LLM (Llama/Qwen) — позже, если объём оправдает

2. **Как шьём знания о нашей системе?**
   - System prompt с философией + правилами + примерами
   - RAG из vault (философия, метрики, модули, спецификации)
   - Fine-tuning на реальных диалогах (позже)

3. **Как шьём данные пользователя?**
   - Снапшот метрик + последние N транзакций + счета + цели в system prompt
   - Tool calls в API приложения (получить детализацию по запросу)

4. **Где сидит чат в UI?**
   - Отдельная страница `/assistant`
   - Плавающая кнопка на дашборде
   - Кнопка «спросить AI» на каждом виджете (контекстный вопрос)

5. **Стоимость и лимиты.**
   - Квота запросов в месяц (по тарифу)
   - Кеширование частых вопросов
   - Rate limits

## Открытые вопросы

1. **Монетизация.** Включено в любой модуль? Отдельная подписка? Квота по тарифам?
2. **Ответственность.** AI советует «досрочно погасить N» — и пользователь пожалел. Как формулируем ограничения?
3. **Чат без данных?** До первой загрузки выписки AI может отвечать только абстрактно. Разрешаем или нет?
4. **История диалогов.** Храним? Пользователь может вернуться? Используем как контекст в следующих вопросах?
5. **Язык.** Только русский на старте?
6. **Интеграция со школьными моментами.** AI может проактивно писать «я заметил у тебя расхождение Базовый vs Полный, хочешь обсудить?» — или только по запросу?
