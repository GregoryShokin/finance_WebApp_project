# Фича — Тегирование целевых денег по счетам
#бэклог #v2 #цели #архитектура
> Виртуальное распределение одной цели по нескольким физическим счетам.
---

## Статус
⚪ Идея на будущее (v2). Не в MVP.

## Суть

Пользователь копит на крупную цель (квартира, 2 млн) и распределяет деньги по нескольким вкладам в разных банках (лимит АСВ 1.4 млн). Система «тегирует» часть баланса каждого счёта как принадлежащую цели.

Пример: цель «Квартира» 2 млн = 1 млн на вкладе Почта Банк + 1 млн на вкладе Райффайзен. Прогресс цели = сумма тегов.

При переводе с вклада на дебетовую — тег перемещается: «такая-то часть денег на карте — целевые».

## Почему не в MVP

1. **Целевая аудитория:** фича нужна семье «Прорыв» (цели > 1.4 млн). Семья «Первый шаг» копит подушку 100к — один счёт.
2. **Архитектурная тяжесть:**
   - Сейчас: прогресс = баланс привязанного счёта. Просто и надёжно.
   - С тегами: прогресс = сумма тегов по N счетам. Рассинхрон с реальным балансом (проценты на вкладе, комиссии).
   - Каждый перевод на любой счёт → вопрос «это на цель?». Больше трения.
   - Снятие с вклада → UI для разделения «сколько целевых, сколько свободных».
3. **Компас миссии:** не помогает семье с DTI 42% и СМО 5 000.

## MVP-замена: школьный момент

Вместо фичи — подсказка: «На цели "Квартира" уже 1.3 млн на одном вкладе. Помни про лимит АСВ 1.4 млн — возможно, стоит открыть второй вклад.»

Пользователь сам распределяет деньги. Система напоминает. Это покрывает 90% потребности без архитектурных изменений.

## Как реализовать (когда придёт время)

- Модель: `GoalAllocation(goal_id, account_id, amount)` — N:N связь цели и счетов
- Прогресс = `Σ(allocations.amount)`, не баланс счёта
- При пополнении счёта с активной целью → вопрос «Сколько из этого на цель?»
- При снятии → вопрос «Снимаешь целевые или свободные?»
- YNAB-подход: деньги физически в одном месте, виртуально — в «конвертах»
