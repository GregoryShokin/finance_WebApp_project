## Допущения и нормализация

**Формулы метрик** (по спецификации):
- **Базовый поток** = регулярные доходы − регулярные расходы (проценты по кредитам = регулярный расход)
- **Свободные средства** = Базовый поток − Σ тел кредитных платежей
- **Полный поток** = Δ баланса ликвидных счетов (regular + cash) за период
- **DTI** = Σ ежемесячных платежей / средний регулярный доход × 100%
- **Капитал** = ликвидные активы + вклады − долги по кредитам
- **Запас** = ликвидный кэш / средний ежемесячный отток (расходы + тела кредитных платежей)

**Нормализация FI-score** (приблизительная, точные кривые не зафиксированы):
- Норма сбережений: `min(10, max(0, (баз.поток / рег.доход) × 25))` → 0%=0, 20%=5, 40%+=10
- Капитал (траектория): экспертная оценка 0–10 по направлению и величине
- DTI (inverse): `max(0, 10 × (1 − DTI/60))` → 0%=10, 30%=5, 60%+=0
- Запас: `min(10, месяцы × 10/6)` → 0=0, 3=5, 6+=10

**Принцип учёта кредитов:**
- Покупка = расход в категории (по регулярности категории)
- Проценты = расход, регулярный
- Тело погашения = трансфер (не расход, не в потоке)
- Выдача кредита = не доход, не расход; в Капитале нетто = 0
- Досрочное погашение = не расход, не в потоке; в Капитале нетто ≈ 0

## Блок A: Кредитные деньги на дебете

### A1. Иллюзия богатства после получения кредита

**Вводные:** Доход 100к (regular), расходы 80к (regular). Потребкредит 500к (12к/мес: 10к тело + 2к %). Деньги на дебете. Стартовый баланс дебета: 20к.

Движения за месяц:
- Доход +100к → дебет 120к
- Выдача кредита (credit_disbursement) +500к → дебет 620к, долг 500к
- Расходы −80к → дебет 540к
- Платёж по кредиту −12к → дебет 528к, долг 490к (500к − 10к тела)

**Расчёт метрик:**
- Базовый поток: 100к − 80к − 2к (%) = **+18к**
- Свободные средства: 18к − 10к (тело) = **+8к**
- Полный поток: Δ дебета = 528к − 20к = **+508к** ← аномалия
- DTI: 12к / 100к = **12%** (норма)
- Капитал: 528к − 490к = **38к** (было 20к → рост +18к, нетто от кредита = 0)
- Запас: 528к / (80к + 12к) = 528к / 92к = **5.7 мес** (норма)
- FI-score: сбережения 18% → 4.5; капитал растёт → 7; DTI 12% → 8; запас 5.7 → 9.5 → **≈ 7.0**

**Вердикт:** ⚠️ Нюанс

**Объяснение:** Запас показывает 5.7 мес — зелёная зона. Но 490к из 528к на дебете — заёмные. Пользователь видит «подушку на полгода», хотя реальный собственный кэш — 38к (Капитал). Полный поток +508к — бессмысленная цифра, раздутая выдачей кредита. Базовый поток (+18к) и Капитал (38к) — честны. Запас и Полный поток — вводят в заблуждение.

**Усиление:** UI-баннер при наличии credit_disbursement в текущем месяце: «На вашем счёте 490к кредитных денег. Запас без них: 0.4 мес». Полный поток — аннотация: «Без кредитных операций: +8к». → Мета-вопросы #1, #2.

### A2. Расход кредитных денег на регулярные нужды

**Вводные:** Тот же пользователь. 5 месяцев тратит 180к/мес (80к обычные + 100к из кредитных денег) на продукты, коммуналку (regular категории). Доход 100к. Стартовый дебет: 528к, долг 490к.

Движения за месяц 1 (из пяти):
- Доход +100к → дебет 628к
- Расходы −180к → дебет 448к
- Платёж по кредиту −12к → дебет 436к, долг 480к

**Расчёт метрик (месяц 1):**
- Базовый поток: 100к − 180к − 2к = **−82к** ← ярко-красный
- Свободные средства: −82к − 10к = **−92к**
- Полный поток: 436к − 528к = **−92к** ← сходится со Свободными (нет нерегулярных)
- DTI: 12к / 100к = **12%** (норма — не ловит проблему!)
- Капитал: 436к − 480к = **−44к** (было +38к → падение 82к)
- Запас: 436к / (180к + 12к) = 436к / 192к = **2.3 мес**
- FI-score: сбережения −82% → 0; капитал падает → 1; DTI 12% → 8; запас 2.3 → 3.8 → **≈ 2.4**

Через 5 месяцев:
- Дебет: 528к − 92к × 5 = **68к**
- Долг: 490к − 10к × 5 = **440к**
- Капитал: 68к − 440к = **−372к**
- Запас: 68к / 192к = **0.35 мес** — критично

**Вердикт:** ✅ Корректно

**Объяснение:** Пользователь говорит «у меня +20к свободных» (100к − 80к). Модель честно показывает −82к: расходы 180к при доходе 100к. Пользователь не осознаёт, что проедает кредитные деньги. Базовый поток, Капитал и Запас — все три ловят проблему уже в первый месяц. FI-score рушится до 2.4.

### A3. Кредитные деньги на целевые расходы (ремонт)

**Вводные:** Кредит 500к на ремонт. Категория «ремонт» = irregular. Тратит 150к/мес × 3 мес + 50к мебель. Доход 100к, регулярные расходы 80к. Стартовый дебет: 528к, долг 490к.

Движения за месяц 1:
- Доход +100к → дебет 628к
- Регулярные расходы −80к → дебет 548к
- Ремонт −150к (irregular) → дебет 398к
- Платёж по кредиту −12к → дебет 386к, долг 480к

**Расчёт метрик (месяц 1):**
- Базовый поток: 100к − 80к − 2к = **+18к** ← ремонт не в базовом (irregular)
- Свободные средства: 18к − 10к = **+8к**
- Полный поток: 386к − 528к = **−142к** ← реальный отток
- DTI: 12к / 100к = **12%**
- Капитал: 386к − 480к = **−94к** (было +38к → падение 132к)
- Запас: 386к / (80к + 150к + 12к) = 386к / 242к = **1.6 мес**
- FI-score: сбережения 18% → 4.5; капитал падает → 2; DTI 12% → 8; запас 1.6 → 2.7 → **≈ 3.8**

**Разница с A2:** Базовый поток ЗЕЛЁНЫЙ (+18к vs −82к) — ремонт в irregular категории изолирован от базовой картины. Полный поток честно показывает −142к (vs −92к в A2, потому что ремонт дороже обычных трат).

**Проект:** Ремонт в irregular категории → механизм Проект = **только UX** (иконка, прогресс-бар). Метрики не корректируются. Это правильно: расходы на ремонт уже исключены из базового потока через irregular.

**Вердикт:** ✅ Корректно

**Объяснение:** Трёхслойный поток работает идеально. Базовый (+18к) показывает устойчивую регулярную картину. Полный (−142к) показывает реальный отток. Разница (150к) = ремонт в irregular. Капитал честно падает. Пользователь видит: «регулярная жизнь в порядке, но ремонт съедает кредитные деньги».

## Блок B: Кредитные карты

### B1. Кешбэк-пользователь (идеальный)

**Вводные:** Доход 200к. Все расходы 120к через кредитку. Полное погашение каждый месяц. На дебете 300к. КК monthly_payment = 0 (нет revolving долга).

Движения:
- Доход +200к → дебет 500к
- Покупки 120к на КК → дебет 500к (не меняется), долг КК +120к
- Погашение КК −120к с дебета → дебет 380к, долг КК 0

**Расчёт метрик:**
- Базовый поток: 200к − 120к (покупки = расходы в категориях) = **+80к**
- Свободные средства: 80к − 0 (monthly_payment = 0) = **+80к**
- Полный поток: 380к − 300к = **+80к** ← все три слоя сходятся!
- DTI: 0 / 200к = **0%**
- Капитал: 380к − 0 = **380к** (рост +80к)
- Запас: 380к / 120к = **3.2 мес** (норма)
- FI-score: сбережения 40% → 10; капитал растёт → 9; DTI 0% → 10; запас 3.2 → 5.3 → **≈ 8.5**

**Вердикт:** ✅ Корректно

**Объяснение:** Идеальный сценарий. DTI = 0 — нет долговой нагрузки. Покупки на КК правильно учтены как расходы в категориях (продукты, транспорт и т.д.). Погашение = трансфер. Свободные средства = Полный поток = Базовый поток = 80к. Три слоя полностью сходятся — нерегулярных аномалий нет.

### B2. Кредитка с revolving-долгом

**Вводные:** Доход 100к. Долг КК 200к. Минимальный платёж 8к/мес (monthly_payment = 8к: 5к тело + 3к %). Новые покупки 15к/мес на КК (regular категории). Обычные расходы 70к с дебета. Стартовый дебет: 100к.

Движения:
- Доход +100к → дебет 200к
- Расходы −70к с дебета → дебет 130к
- Покупки 15к на КК (расход в категориях, дебет не меняется) → долг КК +15к
- Проценты 3к на КК (расход регулярный, начислены на КК) → долг КК +3к
- Платёж с дебета −8к → дебет 122к, долг КК = 200к + 15к + 3к − 8к = **210к**

**Расчёт метрик:**
- Базовый поток: 100к − 70к − 15к (покупки на КК) − 3к (%) = **+12к**
- Свободные средства: 12к − 5к (тело) = **+7к**
- Полный поток: Δ дебета = 122к − 100к = **+22к** ← расхождение!
- DTI: 8к / 100к = **8%** (норма)
- Капитал: 122к − 210к = **−88к** (было 100к − 200к = −100к → рост +12к)
- Запас: 122к / (70к + 15к + 3к + 5к) = 122к / 93к = **1.3 мес** (минимум)
- FI-score: сбережения 12% → 3; капитал растёт но отрицательный → 4; DTI 8% → 8.7; запас 1.3 → 2.2 → **≈ 4.2**

**Проверка связи слоёв:**
- Свободные средства: +7к
- Полный поток: +22к
- Разница: **+15к** = покупки на КК (15к), которые являются расходом в базовом потоке, но НЕ уменьшают дебет. Минус 5к тела, которое уменьшает дебет, но НЕ является расходом. Нетто: 15к − 5к = 10к. Плюс 3к процентов, начисленных на КК (расход, но не с дебета) и 8к платежа с дебета, из которых 3к = проценты (уже в расходах), 5к = тело (не расход). Итого дельта = 15к (покупки не с дебета) − 5к (тело не расход, но с дебета) − 3к (% расход, но не с дебета) + 3к (% в платеже, с дебета но не расход) = 10к... Нет, проще: Полный поток − Свободные = 22к − 7к = 15к. Это расхождение = **нетто кредитная эмиссия** (покупки 15к на КК создают расход без оттока с дебета, а оплата 5к тела — отток с дебета без расхода; 15к − 5к = 10к + 3к% начислены на КК = 13к... Расхождение 15к точно равно новым покупкам на КК).

Это **структурное расхождение** для revolving-кредиток — не аномалия, а характеристика. Каждый рубль покупки на КК = расход (в базовом), но не отток с дебета (в полном). Покупки 15к > погашение тела 5к → долг растёт на 10к/мес.

**Вердикт:** ⚠️ Нюанс

**Объяснение:** Долг растёт на 10к/мес (покупки 15к > тело 5к). Какие метрики ловят?
- ✅ **Капитал** абсолютное значение: −88к (глубоко отрицательный)
- ✅ **Запас**: 1.3 мес (жёлто-красная зона)
- ✅ **Виджет «Траектория долга»**: КК 200к → 210к → рост
- ⚠️ **DTI** = 8% — зелёный, НЕ ловит рост долга (считает только scheduled payment)
- ⚠️ **Базовый поток** = +12к — зелёный, но маскирует рост долга
- ⚠️ **Капитал** тренд = +12к — растёт, маскирует проблему (активы растут быстрее долга)

DTI при revolving-долге с минимальным платежом **системно занижен**. Модель не сломана — DTI честно считает «сколько дохода уходит на обслуживание» — но пользователь может получить ложное спокойствие.

**Усиление:** Школьный момент при росте долга КК ≥ 3 мес подряд: «Долг по кредитной карте вырос на 30к за 3 месяца. При текущем темпе через 2 года долг удвоится. Рассмотрите увеличение платежа с 8к до 20к.»

### B3. Мультикарточный жонглёр

**Вводные:** Доход 150к. Три КК (лимиты 100к, 200к, 300к). Расходы 130к (часть с дебета, часть с КК). Общий долг по КК растёт на 20к/мес. Платежи с дебета: 15к (по 5к на карту). Стартовый дебет: 100к, суммарный долг КК: 300к.

Конструкция: расходы с дебета 100к (regular), покупки на КК 30к (regular), проценты 5к (на КК), переводы между КК — метрически нейтральны.

Движения:
- Доход +150к → дебет 250к
- Расходы −100к с дебета → дебет 150к
- Покупки 30к на КК → долг КК +30к
- Проценты 5к начислены на КК → долг КК +5к
- Платёж −15к с дебета → дебет 135к, долг КК −15к
- Итого долг КК: 300к + 30к + 5к − 15к = **320к** (рост +20к)
- Переводы КК→КК: нетто 0 по долгу

**Расчёт метрик:**
- Базовый поток: 150к − 100к − 30к − 5к = **+15к**
- Свободные средства: 15к − 10к (тела) = **+5к**
- Полный поток: Δ дебета = 135к − 100к = **+35к**
- DTI: 15к / 150к = **10%** (норма)
- Капитал: 135к − 320к = **−185к** (было −200к → рост +15к)
- Запас: 135к / (100к + 30к + 5к + 10к) = 135к / 145к = **0.9 мес** (критично)
- FI-score: сбережения 10% → 2.5; капитал −185к, растёт → 4; DTI 10% → 8.3; запас 0.9 → 1.5 → **≈ 3.9**

**Вердикт:** ⚠️ Нюанс

**Объяснение:** Переводы между КК — метрически нейтральны (трансфер КК→КК не создаёт ни дохода, ни расхода). Это правильно: жонглирование не должно влиять на метрики. Рост общего долга (+20к/мес) **виден** через:
- ✅ Капитал абсолютный: −185к
- ✅ Запас: 0.9 мес (критично)
- ✅ Виджет долга: 300к → 320к → рост
- ⚠️ DTI 10%: зелёный, не ловит
- ⚠️ Капитал тренд +15к: обманчиво положительный

**Усиление:** Школьный момент: «Суммарный долг по 3 кредитным картам вырос на 60к за 3 месяца. Переводы между картами не уменьшают долг — только перемещают его.»

## Блок C: Рассрочки и installment-карты

### C1. Три рассрочки одновременно

**Вводные:** Доход 150к, расходы 90к (regular). Три рассрочки 0%: телефон 60к/12мес = 5к, стиралка 45к/6мес = 7.5к, ноутбук 90к/12мес = 7.5к. Итого платежи 20к/мес. Все покупки были в irregular категориях. Стартовый дебет: 200к, суммарный долг рассрочек: 150к.

Движения (текущий месяц, покупки в прошлых):
- Доход +150к → дебет 350к
- Регулярные расходы −90к → дебет 260к
- Платежи по рассрочкам −20к (всё тело, 0%) → дебет 240к, долг 130к

**Расчёт метрик:**
- Базовый поток: 150к − 90к = **+60к** (рассрочки 0% → нет процентов в расходах)
- Свободные средства: 60к − 20к (тела) = **+40к**
- Полный поток: Δ дебета = 240к − 200к = **+40к** ← сходится!
- DTI: 20к / 150к = **13.3%** (норма)
- Капитал: 240к − 130к = **+110к** (было 200к − 150к = +50к → рост +60к)
- Запас: 240к / (90к + 20к) = 240к / 110к = **2.2 мес** (минимум)
- FI-score: сбережения 40% → 10; капитал растёт → 8; DTI 13% → 7.8; запас 2.2 → 3.7 → **≈ 7.3**

**Вердикт:** ✅ Корректно

**Объяснение:** Три слоя сходятся (Свободные = Полный = 40к, нет нерегулярных). DTI 13.3% правильно суммирует нагрузку трёх рассрочек. Покупки в irregular категориях правильно исключены из текущего базового потока (учтены в месяцы покупки). Свободные средства 40к — реальные деньги после всех обязательств.

### C2. Рассрочка с процентами

**Вводные:** Рассрочка с комиссией 1.5%/мес. Покупка 100к, 10 мес, платёж 11.5к (10к тело + 1.5к %). Доход 150к, расходы 90к. Стартовый дебет: 200к, долг 80к (8 мес осталось).

**Расчёт метрик:**
- Базовый поток: 150к − 90к − 1.5к (%) = **+58.5к**
- Свободные средства: 58.5к − 10к (тело) = **+48.5к**
- Полный поток: Δ дебета = (200к + 150к − 90к − 11.5к) − 200к = **+48.5к** ← сходится!
- DTI: 11.5к / 150к = **7.7%**
- Капитал: (200к + 48.5к) − (80к − 10к) = 248.5к − 70к = **+178.5к** (рост +58.5к)
- Запас: 248.5к / (90к + 1.5к + 10к) = 248.5к / 101.5к = **2.4 мес**
- FI-score: сбережения 39% → 9.8; капитал растёт → 8; DTI 7.7% → 8.7; запас 2.4 → 4.0 → **≈ 7.6**

**Вердикт:** ✅ Корректно

**Объяснение:** Проценты 1.5к правильно учтены как регулярный расход в базовом потоке. Тело 10к — трансфер. Три слоя сходятся. Единственное отличие от 0%-рассрочки: базовый поток ниже на 1.5к, DTI включает проценты. Модель честно отражает стоимость рассрочки.

## Блок D: Множественные кредиты

### D1. Ипотека + потребкредит + рассрочка

**Вводные:** Семейный доход 250к.
- Ипотека: 45к/мес (3к тело + 42к %)
- Потребкредит: 12к/мес (10к тело + 2к %)
- Рассрочка 0%: 5к/мес (5к тело)
- Регулярные расходы: 130к
- Стартовый дебет: 300к. Ипотека: 5М, потреб: 200к, рассрочка: 40к.

**Расчёт метрик:**
- Базовый поток: 250к − 130к − 42к (% ипотеки) − 2к (% потреба) = **+76к**
- Свободные средства: 76к − 3к (тело ипотеки) − 10к (тело потреба) − 5к (рассрочка) = **+58к**
- Полный поток: Δ дебета = +250к − 130к − 45к − 12к − 5к = **+58к** ← сходится!
- DTI: (45к + 12к + 5к) / 250к = 62к / 250к = **24.8%** (допустимо)
- Капитал: (300к + 58к) − (5М − 3к) − (200к − 10к) − (40к − 5к) = 358к − 5222к = **−4864к**
  - Изменение: дебет +58к, долги −18к (3к + 10к + 5к) → Капитал +76к = базовый поток ✅
- Запас: 358к / (130к + 44к + 12к + 5к) = 358к / 191к = **1.9 мес** (минимум)
  - Упрощённо: расходы (130к + 42к + 2к = 174к) + тела (3к + 10к + 5к = 18к) = 192к → 358к / 192к = **1.9 мес**
- FI-score: сбережения 30% → 7.5; капитал отрицательный но растёт → 5; DTI 25% → 5.8; запас 1.9 → 3.2 → **≈ 5.3**

**Проверка связи трёх слоёв:**
1. Базовый поток = 76к (регулярная картина с процентами)
2. Свободные средства = 58к (после всех тел)
3. Полный поток = 58к (реальный Δ кэша)
4. Разница Свободные − Полный = **0** → нет нерегулярных аномалий ✅
5. Разница Базовый − Свободные = **18к** = сумма всех тел кредитов ✅

**Вердикт:** ✅ Корректно

**Объяснение:** Эталонный сценарий. Все три слоя потока идеально сходятся. DTI 24.8% правильно суммирует три кредита. Капитал глубоко отрицательный (ипотека), но растёт на 76к/мес — семья строит благосостояние. Свободные средства 58к — реальный ресурс после всех обязательств.

**Сценарий: «Тихое выгорание подушки»**

**Скрытая проблема:** пользователь подписал договор на покупку квартиры — через 2 месяца нужен первый взнос 2М. Или: работодатель объявил о сокращении через 3 месяца. Или: в семье предстоит дорогое лечение.

Всё это — **внесистемные обязательства**, которых нет в транзакциях.

**Сценарий 2: «Lifestyle inflation»**
- Расходы: янв 80к → фев 90к → мар 100к → апр 110к
- Базовый поток: янв +70к → апр +40к. Всё ещё зелёный, но **тренд нисходящий**

**Сценарий 3: «Монодоход»**
- Один источник (зарплата). Если потеряет — все метрики мгновенно красные
- Модель **не оценивает диверсификацию** дохода → невидимый риск

**Вердикт:** ⚠️ Нюанс

**Объяснение:** Полностью зелёный профиль при реальной проблеме **возможен** в трёх случаях:
1. **Внесистемные обязательства** (договоры, будущие расходы) — за пределами модели
2. **Lifestyle inflation** — модель ловит через тренд базового потока, но не алертит если тренд ещё в зелёной зоне
3. **Монодоходность** — модель не оценивает

Это **не поломка модели**, а фундаментальное ограничение: модель работает с историческими данными, не с будущими обязательствами.

**Усиление:** 
1. Школьный момент при росте расходов > 10% за 3 мес: «Расходы выросли на X%. Если тренд продолжится, свободные средства обнулятся через N мес»

### F4. Досрочное погашение и метрики

**Вводные:** Доход 200к, расходы 100к, потребкредит 500к (12к/мес: 10к тело + 2к %). Дебет 300к. Вносит 200к досрочно.

- Платёж по кредиту −12к → дебет 388к, долг 488к
- Досрочное погашение −200к → дебет 188к, долг 288к

**Расчёт метрик:**

- Свободные средства: 98к − 10к = **+88к**
- Полный поток: Δ дебета = 188к − 300к = **−112к** ← аномалия из-за досрочного
  - Без досрочного: +88к (совпадает со Свободными)
  - Аномалия = −200к (досрочное) → полный поток разово просел
- DTI: 12к / 200к = **6%** (не учитывает досрочное — правильно, это разовое)
- Капитал: 188к − 288к = **−100к** (было: 300к − 500к = −200к → **рост +100к**)
  - Нетто от досрочного: дебет −200к, долг −200к = ±0 ← нейтрально ✅
  - Рост +100к = базовый поток 98к + 2к (округление)
- Запас ДО досрочного: 300к / (100к + 2к + 10к) = 300к / 112к = **2.7 мес**

  - По логике промпта (дебет = 100к в момент сразу после досрочного без дохода): 100к / 112к = **0.9 мес**

**Вердикт:** ✅ Корректно

**Объяснение:** Модель честно показывает trade-off:
- **Капитал ±0** от досрочного: деньги ушли, но и долг уменьшился → нейтрально

- Полный поток −112к — аномален, но **одноразово**; на avg не повлияет через месяц

Это **правильный сигнал**, а не ложный: досрочное погашение действительно снижает подушку безопасности.

**Усиление:** Школьный момент после досрочного погашения: «Запас снизился до 0.9 мес (критично). Рекомендуем восстановить подушку до 3 мес прежде чем делать ещё досрочные погашения. Баланс: безопасность сегодня vs экономия на процентах завтра.»

## Мета-вопросы

### 1. Полный поток после disbursement

**Полный поток = Δ кэша.** При получении 500к кредита кэш вырос на ~508к. Полный поток = +508к. Информативно ли это?

**Ответ: вводит в заблуждение.** Полный поток задуман как «что реально произошло с кэшем» — фактический индикатор. Но credit_disbursement и credit_early_repayment — это не операционный cash flow, а финансовые перемещения. Они искажают Полный поток на порядок.

**Рекомендация:**
- **Вариант А (простой):** Полный поток = Δ кэша, но UI аннотирует: «В этом месяце получен кредит 500к. Операционный поток: +8к»
- **Вариант Б (формульный):** Полный поток = Δ кэша − credit_disbursements + credit_early_repayments. Технически чище, но требует изменения определения
- **Рекомендация:** Вариант А. Полный поток остаётся фактом, аннотация объясняет аномалию. Не ломает определение, не путает пользователя

### 2. Запас и заёмные деньги

**Запас = 528к / 92к = 5.7 мес**, но 490к — заёмные. Должен ли вычитать?

**Ответ: не вычитать из формулы, но обязательно предупреждать.**

Аргументы за вычитание:
- Заёмные деньги целевые (ремонт, авто) — они «не для жизни»
- Запас 5.7 мес создаёт ложное спокойствие

Аргументы против вычитания:
- Технически кредитные деньги МОЖНО потратить на жизнь (A2 это доказал)
- Отслеживание «кредитного конверта» ненадёжно: деньги смешиваются на счёте
- Капитал (38к) уже честно показывает реальность

**Рекомендация:**
1. Формулу не менять: Запас считает ликвидный кэш as-is
2. При наличии credit_disbursement за последние N мес → UI-подсказка: «Из Xк на счёте Yк — кредитные деньги. Запас без них: Z мес»
3. Виджет Запаса показывает вторую линию: «по собственному кэшу» (Запас × доля собственных средств)
4. Капитал рядом с Запасом — пользователь видит оба числа

---

### 3. DTI при нулевом доходе

**Что возвращать?** 

| Ситуация | Числитель | Знаменатель | Результат |
|----------|-----------|-------------|-----------|
| Нет дохода, нет кредитов | 0 | 0 | N/A — нерелевантно |
| Нет дохода, есть кредиты | >0 | 0 | ∞ — критическая ситуация |
| Есть доход, нет кредитов | 0 | >0 | 0% — отлично |

**Рекомендация:**
1. Знаменатель = avg регулярного дохода за 6 мес (не текущий месяц)
2. Если avg за 6 мес = 0 И есть кредиты → показать «Критично: нет регулярного дохода для обслуживания долга»
3. Если avg за 6 мес = 0 И нет кредитов → не показывать DTI (нерелевантно)
4. Никогда не показывать «∞» или «NaN» — это пугает, не помогает

---

### 4. Капитал без истории

**Когда тренд информативен?**

| Месяцев данных | Можно показать | Уверенность |
|---|---|---|
| 1 | Абсолютное значение | Нет тренда |
| 2 | Направление (растёт/падает) | Низкая — одна точка ненадёжна |
| 3+ | Тренд со стрелкой | Достаточная |

**Рекомендация:**
1. 1 месяц: показать число, тренд = «—» (прочерк), подсказка «Для тренда нужно минимум 3 месяца»
2. 2 месяца: показать стрелку (↗/↘) без числового тренда
3. 3+ месяцев: полный тренд, числовая динамика, график
4. В FI-score компонент «Капитал тренд»: при 1 мес → нейтральный вес (5/10); при 2 мес → предварительный; при 3+ → полный

### 5. Нерегулярный доход и DTI

**Бонус 200к (irregular). DTI не меняется, Запас растёт. Это ОК?**

**Ответ: да, это правильное поведение.**

- DTI использует только регулярный доход → бонус не влияет. Это **защита**: если включить бонус в DTI, пользователь увидит «DTI 5%, можно брать ещё кредит», а через месяц бонуса нет → DTI обратно 20%. Ложный сигнал.
- Запас растёт → правильно: кэш на счёте реально увеличился, подушка выросла.
- Капитал растёт → правильно: активы увеличились.
- Базовый поток не меняется → правильно: бонус irregular, не в базовом.
- Полный поток показывает +200к → правильно: факт.

**Единственный нюанс:** если бонус РЕГУЛЯРНЫЙ (ежеквартальный), пользователь должен классифицировать доход как regular. Тогда DTI сгладится через avg за 6 мес. Проблема решается правильной классификацией.

---

### 6. Graceful degradation (все категории = regular)

**Все категории = regular (дефолт). Что теряет ценность?**

| Метрика | Работает? | Комментарий |
|---------|:---------:|-------------|
| Базовый поток | ⚠️ | Включает разовые расходы (ремонт, авто) → завышен/занижен |
| Свободные средства | ⚠️ | Зависит от Базового потока → то же |
| Полный поток | ✅ | Факт, не зависит от классификации |
| DTI | ✅ | Зависит от Account.monthly_payment, не от категорий |
| Капитал | ✅ | Балансовый, не зависит от классификации |
| Запас | ✅ | Балансовый + avg outflow (факт) |
| FI-score | ⚠️ | Компонент «Норма сбережений» искажён (зависит от Базового потока) |
| Аналитика расходов | ❌ | Нет разделения regular/irregular — бесполезна |
| Показатель устойчивости | ⚠️ | Раздут разовыми расходами |

**Итог:** 3 из 4 основных метрик работают нормально (DTI, Капитал, Запас). Базовый поток и зависимые от него показатели теряют ценность. Аналитика расходов бесполезна без классификации.

**Деградация мягкая:** пользователь получает полезные сигналы даже без настройки. Но при первом крупном нерегулярном расходе (ремонт 200к) базовый поток рухнет → школьный момент: «Базовый поток упал на 200к. Отметьте разовые расходы как нерегулярные для точности показателей.»

### Ключевые выводы:

**Ни одна формула не сломана.** Все 18 сценариев прошли без критических поломок. Модель не даёт опасных советов и не врёт больше чем на 20% ни в одной метрике.

**Системная слабость #1: кредитные деньги на дебете** (A1, F3, мета #1, #2). Запас и Полный поток раздуваются при получении кредита. Капитал корректирует, но пользователь может не сопоставить. Решение: UI-аннотации + вторая линия Запаса «по собственному кэшу».

**Системная слабость #2: DTI при revolving-долге** (B2, B3). DTI считает только scheduled payment, не видит рост долга от новых покупок. Решение: виджет «Траектория долга» + школьные моменты. Не требует изменения формулы DTI.

**Системная слабость #3: чувствительность к длине окна** (E2). При сезонном доходе 3-месячное окно даёт ложные сигналы. Решение: авторасширение окна при высокой дисперсии.

**Что работает безупречно:**
- Трёхслойный поток: связь Базовый → Свободные → Полный сходится во всех сценариях
- Кредитная логика: disbursement, досрочное погашение, рефинансирование — всё нейтрально в Капитале
- Принцип «покупка = расход»: кешбэк-пользователь, revolving, рассрочки — единая модель для всех
- Irregular/regular разделение: ремонт правильно изолирован от базового потока
