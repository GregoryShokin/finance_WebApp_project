## Мой ответ на открытые вопросы

### 1. Дашборд в ремонтные месяцы — концепт

Пользователь: доход 200к, обычные расходы 105к, кредит 500к на ремонт (проект, 3 мес).

**Верхняя панель — четыре карточки:**

```
┌──────────────┐  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│  ПОТОК       │  │  КАПИТАЛ     │  │  НАГРУЗКА    │  │  ЗАПАС       │
│  +95к  ✅    │  │  ↓ −60к/мес  │  │  0%     ✅   │  │  0.9 мес 🔴  │
│  (−55к всего)│  │  ⚠️ плановый │  │              │  │              │
│  📋 Ремонт  │  │              │  │              │  │              │
└──────────────┘  └──────────────┘  └──────────────┘  └──────────────┘
```

Логика отображения:
- **Поток**: главное число = базовый поток (+95к, зелёный). Под ним мелко: полный поток (−55к). Иконка 📋 = активный проект. Клик → детали проекта.
- **Капитал**: траектория ↓, но с пометкой «плановый» (проект активен).
- **Нагрузка**: 0% — платежей ещё нет.
- **Запас**: честный, 0.9 мес. Красный. Это правда — ликвидных мало.

**FI-score** считается по: базовый поток (+95к) + траектория капитала (↓, но «плановый») + нагрузка (0%) + запас (0.9 мес). Результат: ~6/10 (запас тянет вниз, остальное ок). Без проекта было бы ~4/10 (поток −55к тоже красный).

**Школьный момент (триггер: активный проект + запас < 1 мес):**
> «У тебя активный проект "Ремонт" — расходы идут по плану. Базовый поток +95к — уровень жизни не изменился. Но запас упал до 0.9 мес — следи за ним. Когда ремонт закончится, поток восстановится и запас начнёт расти.»

**Месяц 4+ (ремонт закончен, платежи пошли):**

```
┌──────────────┐  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│  ПОТОК       │  │  КАПИТАЛ     │  │  НАГРУЗКА    │  │  ЗАПАС       │
│  +87к  ✅    │  │  ↑ +80к/мес  │  │  22.5%  🟡   │  │  1.2 мес 🟡  │
│              │  │  🟢          │  │              │  │  ↑ растёт    │
└──────────────┘  └──────────────┘  └──────────────┘  └──────────────┘
```

Проект закрыт. Все метрики честные. Нагрузка появилась (22.5%). Запас восстанавливается.

### 3. Edge cases проекта

- **Бюджет перерасходован:** проект автоматически НЕ закрывается при достижении бюджета. Но школьный момент: «Расходы на "Ремонт" превысили план на 50к. Пересмотреть бюджет проекта?» Пользователь решает: увеличить бюджет или закрыть проект.
- **Закончился раньше:** пользователь вручную закрывает проект. Или: если расходы прекратились (0 за месяц) — система предлагает закрыть.
- **Несколько проектов одновременно:** каждый с отдельным бюджетом и сроком. Базовый поток = полный поток − Σ(бюджет_i / срок_i). Не ограничиваем количество, но школьный момент при >2: «У тебя 3 активных проекта — твой базовый поток может быть менее точным.»
