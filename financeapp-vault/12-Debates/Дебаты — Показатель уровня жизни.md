# Дебаты: Показатель уровня жизни

## Участники
- **АРХИТЕКТОР** — предлагает формулу, пороги, визуализацию
- **КРИТИК** — атакует каждое предложение конкретными сценариями с цифрами

## Контекст

Мультиметрическая система (принята):
- **Поток** — двухслойный: базовый (регулярные) + полный (delta кэша)
- **Капитал** — активы + кэш − долги
- **Нагрузка (DTI)** — платежи по кредитам / регулярный доход
- **Запас** — доступный кэш / средний ежемесячный отток (в месяцах)
- **FI-score** — композитный балл

Базовый поток = регулярные доходы − регулярные расходы − платежи по кредитам (полный платёж: тело + проценты).

Задача: найти формулу показателя, отвечающего на вопрос **«Может ли пользователь позволить себе текущий уровень жизни?»**

---

## Раунд 1: Формула и базовое определение

### Архитектор

Предлагаю: **Показатель устойчивости образа жизни** — нормализованный базовый поток.

**Формула:**

```
Показатель = Базовый поток / Регулярный доход × 100%
```

Где:
- Базовый поток = Σ(регулярные доходы) − Σ(регулярные расходы) − Σ(платежи по кредитам)
- Регулярный доход = Σ(доходы с флагом `is_regular = true`)

**Почему нормализация, а не абсолютное число:**
- «+80к свободно» ничего не значит без контекста дохода
- «40% дохода свободно» — сразу ясно
- Процент сравним между пользователями с разным доходом

**Что видит пользователь:**
- Процент + абсолютная сумма: **«Свободно 40% дохода (80 000 ₽/мес)»**
- При дефиците: **«Дефицит −8% (−10 000 ₽/мес)»**

**Сглаживание:** скользящее среднее за 3 месяца по обоим компонентам:

```
Показатель = avg(базовый_поток, 3 мес) / avg(рег_доход, 3 мес) × 100%
```

При данных менее 3 месяцев — считаем по доступным (1–2 месяца) с пометкой «Мало данных для надёжной оценки».

### Критик

**Атака 1: Сценарий 1 — Здоровая семья**
- Доход 200к, расходы 120к (рег.), кредитов нет
- Показатель = (200 − 120 − 0) / 200 = 80/200 = **40%**
- Однозначно зелёный. ✅ Держится.

**Атака 2: Сценарий 2 — Живёт впритык**
- Доход 120к, расходы 115к (рег.), кредитов нет
- Показатель = (120 − 115) / 120 = 5/120 = **4.2%**
- Формально положительный. Но 5к — это один поход в магазин. Какая зона? Если жёлтая — ОК, если зелёная — показатель врёт. → Вопрос к раунду 2 (пороги).

**Атака 3: Деление на ноль**
- Фрилансер в месяц без выплат. Регулярный доход = 0.
- Показатель = −120к / 0 = **undefined**.
- Нужна защита. Что показываем?

**Атака 4: Почему именно 3-месячное сглаживание?**
- При 3 месяцах: январь 300к, февраль 80к, март 80к → avg = 153к. Через месяц: февраль 80к, март 80к, апрель 120к → avg = 93к. Скачок среднего при выпадении аномального месяца.
- При 6 месяцах: слишком медленная реакция на реальное падение дохода (декрет, увольнение).
- 3 месяца — компромисс, но не идеальный. Достаточно ли?

### Решение раунда
- **Принято:** Формула = Базовый поток / Регулярный доход × 100%. Показывать и процент, и абсолютную сумму.
- **Принято:** Сглаживание 3 месяца. Компромисс между реактивностью и стабильностью. При < 3 мес данных — считать по доступным с предупреждением.
- **Принято:** При регулярном доходе = 0 (все 3 месяца) — специальное состояние «Нет данных о регулярном доходе». Показатель не рассчитывается.
- **Открыто:** Пороги зон → Раунд 2.

## Раунд 2: Пороги и зоны

### Архитектор

Предлагаю три зоны:

| Зона | Показатель | Интерпретация |
|------|-----------|---------------|
| 🟢 Зелёная | ≥ 20% | Здоровый запас, есть на сбережения |
| 🟡 Жёлтая | 0% – 19% | Справляется, но запас мал |
| 🔴 Красная | < 0% | Живёт не по средствам |

**Логика порогов:**
- **20%** — правило 50/30/20: 20% дохода на сбережения. Если после обязательств остаётся ≥ 20% — базовый уровень финансового здоровья.
- **0%** — математический порог. Отрицательный = расходы > доходов = живёт не по средствам. Чёткий, объективный, не спорный.
- **Не делю красную зону на подзоны** (−5% vs −50%). Степень проблемы показывает абсолютная сумма дефицита.

### Критик

**Атака 1: Сценарий 4 — Семья «Первый шаг»**
- Доход 120к, расходы 65к, кредиты 50к/мес
- Показатель = (120 − 65 − 50) / 120 = 5/120 = **4.2%**
- Жёлтая зона. Эта семья платит 50к по кредитам и умудряется не уходить в минус. Они не «живут не по средствам» — они борются с долгом. Жёлтая корректна.
- Но тот же 4.2% у семьи без кредитов (сценарий 2) — совсем другая ситуация. Показатель одинаков, причина разная.
- **Контраргумент самому себе:** DTI разделяет: сценарий 2 → DTI 0%, сценарий 4 → DTI 42%. Вместе метрики дают полную картину. Показатель не обязан различать причину сам по себе.
- ✅ Держится в контексте мультиметрики.

**Атака 2: Абсолютная недостаточность при низком доходе**
- Доход 50к, расходы 40к → показатель = 20%. Зелёный!
- Но 10к свободных — это критически мало в абсолюте. Один визит к стоматологу = минус.
- Показатель «врёт»?
- **Нет.** Показатель отвечает на конкретный вопрос: «можешь ли позволить ТЕКУЩИЙ уровень жизни?» Ответ: да, структурно можешь (20% запас). Что 10к мало для форс-мажоров — покажет **Запас** (сколько месяцев протянешь на накоплениях). Каждая метрика отвечает на свой вопрос.
- ✅ Не баг, а разделение ответственности между метриками.

**Атака 3: Сценарий 3 — Не по средствам**
- Доход 120к, расходы 130к (рег.), дефицит закрывает кредиткой
- Показатель = (120 − 130) / 120 = **−8.3%**
- Красный. Корректно. Дефицит 10к/мес — ясно показан.
- ✅ Работает точно как задумано.

**Атака 4: Граница 0% — слишком мягкая?**
- Показатель +1% (1.2к при доходе 120к) = жёлтый, «справляется».
- Но 1.2к свободных — это практически то же, что 0. Один лишний ужин в ресторане и ты в минусе.
- Может, красная должна быть < 5%?
- **Контраргумент:** раздвигая красную зону, мы лишаем семьи вроде «Первый шаг» (4.2%) заслуженной жёлтой. Они СПРАВЛЯЮТСЯ. Показатель +4.2% = «ты в плюсе, но впритык». Если сделать < 5% красным, то семья, которая героически платит по долгам и остаётся в плюсе, увидит «живёт не по средствам» — а это неправда.
- Граница 0% = математически честная. Ноль есть ноль.

### Решение раунда
- **Принято:** Три зоны: 🟢 ≥ 20%, 🟡 0–19%, 🔴 < 0%.
- **Принято:** Красная = строго отрицательный показатель. Это чёткий, объективный критерий: расходы > доходов.
- **Принято:** Абсолютная сумма всегда рядом с процентом — пользователь сам оценивает достаточность.
- **Отвергнуто:** Красная < 5% — ложные алармы для семей, которые справляются.
- **Отвергнуто:** Подзоны красного — сумма дефицита достаточна для понимания степени проблемы.
- **Подтверждено:** Одинаковый показатель при разных причинах (сценарии 2 и 4) — не баг. DTI дифференцирует.

## Раунд 3: Кредитная нагрузка в показателе

### Архитектор

Кредитные платежи уже в формуле как полный ежемесячный платёж (тело + проценты). Разбор по типам:

| Тип кредита | Что на счёте | Что в базовом потоке |
|-------------|-------------|---------------------|
| **Рассрочка** | Ежемесячный платёж | Платёж (кредитный) |
| **Потребкредит** | Disbursement (нерег. доход) + расходы из кредитных денег (нерег.) + ежемесячный платёж | Только платёж (нерег. взаимогасятся) |
| **Ипотека / автокредит** | Ежемесячный платёж | Платёж (кредитный) |

Все три типа: в базовый поток попадает ТОЛЬКО ежемесячный платёж. Покупки за кредитные деньги и получение кредита — нерегулярные операции, не влияют на показатель.

### Критик

**Атака 1: Сценарий 5 — Ипотека + автокредит**
- Доход 200к, расходы 80к, ипотека 40к, автокредит 20к
- Показатель = (200 − 80 − 40 − 20) / 200 = 60/200 = **30%** → 🟢
- DTI = (40 + 20) / 200 = **30%**
- Зелёный по показателю, но DTI 30% — существенная нагрузка.
- **Конфликт?** Нет. Показатель: «ты можешь позволить этот образ жизни, включая кредиты». DTI: «30% дохода уходит банкам — при потере дохода будет больно». Разные вопросы, оба ответа верны.
- ✅ Держится.

**Атака 2: Сценарий 6 — Рассрочки накопились**
- Доход 150к, расходы 100к, три рассрочки по 10к = 30к
- Показатель = (150 − 100 − 30) / 150 = 20/150 = **13.3%** → 🟡
- DTI = 30/150 = **20%**
- Жёлтый: справляется, но 20к свободных — маловато.
- Через год рассрочки закончатся → показатель = (150 − 100) / 150 = **33%** → 🟢
- Показатель отражает ТЕКУЩЕЕ состояние и автоматически улучшится. Корректно.
- ✅ Держится.

**Атака 3: Математическая связь КУЖ и DTI**
- Показатель = 1 − (Рег. расходы / Рег. доход) − DTI
- Т.е.: Показатель = 1 − Expense Ratio − DTI
- Кредитные платежи участвуют и в показателе (как вычитаемое), и в DTI (как числитель).
- **Дублирование?** Нет. DTI изолирует кредитную составляющую: «сколько отдаёшь банкам». Показатель — итоговая картина: «сколько остаётся ВООБЩЕ». Это компоненты одной системы, не дублирование.
- Аналогия: DTI — часть уравнения, показатель — весь ответ.

### Решение раунда
- **Принято:** Все типы кредитов корректно обработаны. Полный платёж (тело + проценты) в базовом потоке.
- **Принято:** Показатель и DTI — комплементарные метрики. Математически связаны (Показатель = 1 − ExpRatio − DTI), но отвечают на разные вопросы.
- **Подтверждено:** Показатель — моментальный снимок. Закончится кредит → показатель улучшится автоматически.
- **Подтверждено:** Сценарии 5 и 6 проходят без проблем.

## Раунд 4: Нерегулярные и маскировка

### Архитектор

Базовый поток по определению исключает нерегулярные доходы и расходы. Это ключевое свойство:

1. **Бонусы не маскируют дефицит** — нерегулярный доход не в базовом потоке
2. **Крупные покупки не роняют показатель** — нерегулярный расход не в базовом потоке
3. **Показатель = структурная устойчивость** — «можешь жить так без бонусов и без крупных трат?»

Для полной картины есть полный поток (delta кэша за период). Но показатель основан на базовом — это его сила.

**Дополнение:** подсказка при значительном расхождении базового и полного потока.

Триггер: средние нерегулярные расходы за 3 месяца > 30% от регулярного дохода.

Текст: *«Ваш базовый образ жизни устойчив, но нерегулярные расходы значительны (X₽/мес). Проверьте, не стали ли некоторые из них регулярными.»*

### Критик

**Атака 1: Сценарий 7 — Бонус маскирует проблему**
- Регулярный доход 100к, расходы 110к (рег.)
- Квартальный бонус 150к (нерегулярный)
- Показатель = (100 − 110) / 100 = **−10%** → 🔴

Красный. Показатель чётко видит: регулярные расходы > регулярных доходов. Бонус не маскирует.

Пользователь может возразить: «Но я же не в минусе!» Приложение отвечает: *«Ваши регулярные расходы превышают регулярный доход на 10к/мес. Квартальный бонус покрывает дефицит, но это ненадёжный источник. Если бонус прекратится — дефицит станет постоянным.»*

✅ **Держится отлично.** Это exactly то, для чего нужен показатель.

**Атака 2: Сценарий 8 — Нерегулярные расходы давят**
- Доход 200к, рег. расходы 100к, нерегулярные 80к/мес (хобби, подарки, мебель)
- Показатель = (200 − 100) / 200 = **50%** → 🟢
- Полный поток = 200 − 100 − 80 = 20к
- Показатель зелёный, но реально свободно только 20к.

**Это баг или фича?**

**Фича**, если нерегулярные расходы действительно нерегулярные (можно сократить).
**Баг**, если «нерегулярные» 80к/мес — на самом деле ежемесячные хобби, которые пользователь ошибочно пометил нерегулярными.

Формула верна при правильной классификации. Проблема — в данных, не в формуле.

**Но:** подсказка срабатывает? Средние нерег. расходы = 80к, доход 200к → 80/200 = 40% > 30% → **ДА, подсказка сработает.** Пользователь увидит: «Нерегулярные расходы 80к/мес — проверьте, не стали ли регулярными.»

✅ **Держится с подсказкой.**

**Атака 3: Что если нерегулярных мало, но они ВСЕГДА есть?**
- Каждый месяц 15к нерегулярных расходов (разные категории: подарки, такси-на-дачу, ветеринар)
- 15/200 = 7.5% < 30% → подсказка НЕ сработает
- Но 15к × 12 = 180к/год — существенная сумма

**Это нормально.** 15к нерегулярных при 200к доходе — не критично. Показатель 50%, полный поток 42.5% — оба зелёные. Если пользователя это беспокоит — он сам пересмотрит классификацию. Система не должна волноваться за него при 7.5% нерегулярных.

✅ Порог 30% адекватен.

### Решение раунда
- **Принято:** Базовый поток правильно изолирует нерегулярные. Показатель = структурная устойчивость.
- **Принято:** Сценарий 7 — бонус не маскирует. Красный. Идеальная работа.
- **Принято:** Сценарий 8 — зелёный по базовому + подсказка о нерегулярных. Корректно.
- **Принято:** Подсказка при нерегулярных расходах > 30% регулярного дохода (среднее за 3 мес).
- **Отвергнуто:** «Полный показатель» (на основе полного потока) — два показателя избыточны и путают.
- **Подтверждено:** Точность показателя зависит от правильной классификации пользователем. Подсказка помогает скорректировать ошибки.

## Раунд 5: Взаимодействие с мультиметрикой

### Архитектор

Ключевое утверждение: **показатель устойчивости — это пользовательское представление метрики «Поток», а не отдельная пятая метрика.**

| Метрика | Вопрос | Связь с показателем |
|---------|--------|-------------------|
| **Поток (= Показатель)** | Можешь позволить текущий образ жизни? | Это он и есть |
| **Капитал** | Сколько у тебя чистого состояния? | Не связан: можно проедать капитал при красном потоке |
| **DTI** | Какая доля дохода уходит банкам? | Компонент показателя: Показатель = 1 − ExpRatio − DTI |
| **Запас** | Сколько месяцев протянешь без дохода? | Дополняет: даже при красном потоке, большой запас = есть время исправить |
| **FI-score** | Общая оценка здоровья | Показатель входит как компонент (скользящее среднее базового потока) |

**Что это значит для дашборда:**
- Пользователь видит 4 метрики + FI-score (как было в мультиметрике)
- Виджет «Поток» показывает: процент + сумма + зона (зелёный/жёлтый/красный) + школьный момент
- Внутреннее имя: «базовый поток». Пользовательское имя: обсуждаемо, но это не вопрос формулы

### Критик

**Атака 1: Если показатель = просто нормализованный поток, зачем дебаты?**

Базовый поток уже определён в мультиметрике. Мы добавили:
1. Нормализацию (÷ доход) → процент
2. Сглаживание (3 мес)
3. Зоны (🟢🟡🔴)
4. Подсказку по нерегулярным
5. Школьные моменты

Это не «просто поделили на доход». Это превращение сырого числа в осмысленный показатель с интерпретацией. Дебаты валидировали каждый элемент.

✅ Дебаты оправданы.

**Атака 2: Сценарий 9 — Фрилансер с волатильным доходом**
- Доход: январь 150к, февраль 80к, март 300к, апрель 120к
- Расходы: стабильные 120к/мес, кредитов нет

По месяцам:
| Месяц | Доход | Расход | Базовый поток | Показатель (месяц) |
|-------|-------|--------|--------------|-------------------|
| Январь | 150к | 120к | +30к | +20% |
| Февраль | 80к | 120к | −40к | −50% |
| Март | 300к | 120к | +180к | +60% |
| Апрель | 120к | 120к | 0 | 0% |

Без сглаживания: показатель скачет от −50% до +60%. Бесполезно.

Со сглаживанием (3 мес):
- На конец марта: avg_поток = (30−40+180)/3 = 56.7к, avg_доход = (150+80+300)/3 = 176.7к → **32%** → 🟢
- На конец апреля: avg_поток = (−40+180+0)/3 = 46.7к, avg_доход = (80+300+120)/3 = 166.7к → **28%** → 🟢

### Итоговая формула

```
Показатель устойчивости = avg(Базовый поток, 3 мес) / avg(Регулярный доход, 3 мес) × 100%
```

Где:
- **Базовый поток** = Σ(регулярные доходы) − Σ(регулярные расходы) − Σ(платежи по кредитам)
- **Регулярные** = транзакции с `is_regular = true` (snapshot флага категории)
- **Платежи по кредитам** = полный ежемесячный платёж (тело + проценты), всегда в базовом потоке
- **Сглаживание** = скользящее среднее за 3 месяца. При < 3 мес данных — по доступным с пометкой

### Что видит пользователь

**Формат:** процент + абсолютная сумма.

| Зона | Показатель | Пример отображения |
|------|-----------|-------------------|
| 🟢 Зелёная | ≥ 20% | «Свободно 40% дохода (80 000 ₽/мес)» |
| 🟡 Жёлтая | 0% – 19% | «Свободно 4% дохода (5 000 ₽/мес)» |
| 🔴 Красная | < 0% | «Дефицит −8% (−10 000 ₽/мес)» |

При расхождении базового и полного потока (нерег. расходы > 30% дохода):
> «Обратите внимание: нерегулярные расходы составляют ~80 000 ₽/мес. Базовый образ жизни устойчив, но нерегулярные траты значительно сокращают свободные средства.»

### Пороги

| Зона | Порог | Критерий |
|------|-------|----------|
| 🟢 Зелёная | ≥ 20% | Есть запас на сбережения (правило 50/30/20) |
| 🟡 Жёлтая | 0% – 19% | Положительный, но мал — уязвим к форс-мажорам |
| 🔴 Красная | < 0% | Отрицательный = расходы > доходов = не по средствам |

**Подсказка по нерегулярным (любая зона):**

### Взаимодействие с мультиметрикой

```
Показатель (Поток)  → «Можешь позволить текущий образ жизни?»
Капитал             → «Сколько у тебя чистого состояния?»
DTI                 → «Какая доля дохода уходит банкам?»
Запас               → «Сколько месяцев протянешь без дохода?»
FI-score            → «Общая оценка финансового здоровья»
```

- Показатель = пользовательское представление метрики «Поток» (не отдельная пятая метрика)
- DTI — компонент показателя (Показатель = 1 − ExpenseRatio − DTI)
- Запас дополняет при красном показателе: «дефицит X, но накоплений хватит на Y месяцев»
- FI-score использует абсолютный базовый поток (скользящее среднее), не процент

## Реестр решений

### Принято
1. **Формула:** Показатель = avg(Базовый поток, 3 мес) / avg(Регулярный доход, 3 мес) × 100%
2. **Связь с базовым потоком:** показатель = нормализованный базовый поток. Это не новая метрика, а способ показать Поток пользователю.
3. **Формат:** процент + абсолютная сумма всегда вместе
4. **Три зоны:** 🟢 ≥ 20% / 🟡 0–19% / 🔴 < 0%
5. **Красная = только отрицательный.** Чёткий математический критерий: расходы > доходов.
6. **Сглаживание 3 месяца.** Компромисс между реактивностью и стабильностью.
7. **Подсказка по нерегулярным** при расхождении > 30% дохода.
8. **Школьные моменты** для каждой зоны — конкретные, с цифрами, с советом.
9. **Показатель и DTI — комплементарны** (Показатель = 1 − ExpRatio − DTI).
10. **При нулевом доходе** — показатель не рассчитывается, специальное состояние.

### Отвергнуто
1. **«Полный показатель»** (на основе полного потока) — два показателя путают пользователя. Подсказка достаточна.
2. **Подзоны красного** (−5% vs −50%) — абсолютная сумма дефицита показывает степень.
3. **Красная < 5%** — ложные алармы для семей, которые справляются.
4. **Income-dependent thresholds** — усложнение без значительной выгоды. Абсолютная сумма компенсирует.
5. **Отдельная пятая метрика** — показатель = пользовательский вид Потока.

- Конкретное название показателя в UI (рабочее: «Поток» или «Устойчивость»)
- Визуальный дизайн виджета
- Полная формула FI-score с весами компонентов
