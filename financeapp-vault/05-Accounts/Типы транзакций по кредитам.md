# Типы транзакций по кредитам

#счета #логика #кредиты

> **Актуально на:** 2026-04-16 | Источник: -> [[Спецификация — Целевое состояние системы]]

> Пять типов операций по кредитам. Тип `credit_payment` **упразднён** — заменён комбинацией `expense` (проценты) + `transfer` (тело).

---

## Таблица операций

| Операция | Тип (`operation_type`) | В расходной аналитике | В базовом [[Поток|потоке]] | В «[[Поток|Свободных средствах]]» |
|---------|-----|:---:|:---:|:---:|
| Покупка (в категории) | `expense` | Да | Если `is_regular` | Через базовый поток |
| Поступление кредита | `credit_disbursement` | Нет | Нет | Нет |
| Погашение тела | `transfer` | Нет | Нет | **Тело вычитается** (если обязательный кредитный платёж) |
| Уплата процентов | `expense` | Да | Да (регулярный расход) | Через базовый поток |
| Досрочное погашение | `credit_early_repayment` | Нет | Нет | Нет |

---

## Принцип

**Кредит = отдельный мир.** Только покупка и проценты создают настоящий расход. Всё остальное — перемещения между счетами. См. [[Кредитная логика#Принцип кредит = отдельный мир]].

---

## Детали по типам

### expense (покупка)

Покупка на кредитные средства = расход в реальной категории. Регулярность определяется категорией, не способом оплаты.

| Покупка | Категория | В базовом потоке? |
|---------|-----------|:---:|
| Продукты на кредитку | Продукты (regular) | Да |
| MacBook на кредитку | Электроника (irregular) | Нет |
| Ремонт из потребкредита | Ремонт (project) | Нет (с коррекцией) |

### expense (проценты)

Проценты по кредитам = обычный `expense` в категории «Проценты по кредитам» с флагом `regular`. Входит в базовый поток как регулярный расход.

### credit_disbursement

Получение кредитных средств на счёт. **НЕ доход** (ни регулярный, ни нерегулярный). НЕ появляется в аналитике доходов. Влияет на балансы счетов (дебет увеличивается, долг увеличивается). В [[Капитал|Капитале]]: нетто = 0.

### transfer (тело)

Погашение тела кредита = перевод между счетами. Не расход. Не в базовом потоке. Но **вычитается из Свободных средств** как обязательный отток.

### credit_early_repayment

Досрочное погашение = перевод (тело). Не в расходной аналитике, не в потоке. Влияет на [[Капитал]] и [[DTI]].

## Полная таблица operation_type

| Тип | Описание | В расходной аналитике | В базовом потоке | В «Свободных средствах» |
|-----|---------|:---:|:---:|:---:|
| `income` | Любой доход | -- | Если is_regular | Через базовый поток |
| `expense` | Любой расход (включая покупки на кредит и проценты) | Да | Если is_regular | Через базовый поток |
| `transfer` | Перевод между счетами (в т.ч. тело кредита) | Нет | Нет | **Тело вычитается** (если обязательный) |
| `credit_disbursement` | Получение кредитных средств. **НЕ доход.** | Нет | Нет | Нет |
| `credit_early_repayment` | Досрочное погашение | Нет | Нет | Нет |
| `investment_buy` | Покупка актива | Нет | Нет | Нет |
| `investment_sell` | Продажа актива | Нет | Нет | Нет |
| `debt_give` / `debt_take` / `debt_return` / `debt_collect` | Личные долги | Нет | Нет | Нет |

> [!warning] credit_payment упразднён
> `credit_payment` как отдельный `operation_type` **упразднён** (апрель 2026). Проценты = обычный `expense` в категории «Проценты по кредитам» (регулярная). Тело = `transfer`. Это устраняет необходимость специального типа и решает проблему кешбэк-пользователя.
