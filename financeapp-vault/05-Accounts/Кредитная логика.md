# Кредитная логика

#счета #логика #кредиты

> **Актуально на:** 2026-04-16 | Источник: -> [[Спецификация — Целевое состояние системы]]

> Единая модель учёта кредитов. Все показатели по кредитам считаются из реальных транзакций, не по формулам.

---

## Принцип: кредит = отдельный мир

Только два типа событий создают настоящий расход:
1. **Покупка** — решение потратить деньги (в реальной категории)
2. **Проценты** — плата банку за услугу (регулярный расход)

Всё остальное (получение кредита, возврат тела, досрочное погашение) — **перемещения между счетами**, не доход и не расход.

> [!important] Единая модель
> **Покупка на кредитные средства = расход в реальной категории. Всегда. Для всех типов кредитов.** Никакого выбора режима пользователем. Режим «платёж = расход» отвергнут — ломает кешбэк-сценарий.

---

## Типы операций

-> [[Типы транзакций по кредитам]]

| Операция | Тип | В потоке? |
|---------|-----|:---------:|
| Покупка (расход в категории) | `expense` | Да (по регулярности категории) |
| Поступление кредита на дебет | `credit_disbursement` | Нет |
| Погашение тела | `transfer` | Нет |
| Уплата процентов | `expense` | Да (регулярный расход) |
| Досрочное погашение | `credit_early_repayment` | Нет |

> [!warning] credit_payment упразднён
> Тип `credit_payment` как отдельный `operation_type` **упразднён**. Проценты = обычный `expense` в категории «Проценты по кредитам» (регулярная). Тело = `transfer`. Это устраняет необходимость специального типа.

---

## Правило «каждый рубль — один раз»

Если покупка уже учтена как расход -> тело погашения = перевод (не дублируется в потоке).

## Применение к типам кредитов

### Кредитная карта

```
Покупка 5 000 «Продукты» на кредитку:
  -> Расход -5 000 на счёте КК, категория «Продукты»
  -> Долг на КК: +5 000
  -> Никакого disbursement

Погашение кредитки 5 000 с дебета:
  -> Перевод: дебет -> КК
  -> Долг КК: -5 000
  -> Не расход, не в потоке
```

Кешбэк-пользователь (все 120к/мес через кредитку, полное погашение):
- Все расходы видны по категориям
- DTI = 0% (погашение = перевод)
- Базовый поток = 200к - 120к = 80к

### Карта рассрочки

```
Покупка телефона 60к на рассрочку (12 мес, 0%):
  -> Расход -60 000, категория «Электроника» (нерегулярная)
  -> Долг: +60 000

Ежемесячный платёж 5 000:
  -> Перевод: дебет -> installment_card (тело 5 000)
  -> Если есть проценты: расход (проценты)
  -> Долг: -5 000
```

### Потребительский кредит

```
Получение 500к на дебетовый счёт:
  -> credit_disbursement +500 000 (НЕ доход, НЕ в аналитике доходов)
  -> Кредитный счёт: долг +500 000

Расходование (ремонт 150к/мес, 3 месяца):
  -> Обычные расходы в категориях («Ремонт» — нерегулярная)

Ежемесячный платёж 45к:
  -> Перевод (тело ~40к) + расход (проценты ~5к)
```

### Автокредит / ипотека

```
Ипотека 5М:
  -> credit_disbursement +5 000 000 на дебет
  -> Покупка квартиры -5 000 000 (расход, нерегулярный)
  -> Нетто на дебете: 0. Долг: +5 000 000

Ежемесячный платёж 45к:
  -> Перевод (тело ~3к) + расход (проценты ~42к)
```

## Сводная таблица: кредиты в метриках

| Тип кредита | Покупка | Disbursement | Тело платежа | Проценты | Тело в «Свободных средствах» |
|-------------|---------|:---:|:---:|:---:|:---:|
| Кредитная карта | Расход (категория) | Нет | Перевод | Расход (рег.) | Нет (погашение = разовый перевод) |
| Карта рассрочки | Расход (категория) | Нет | Перевод | Расход (= 0) | **Да** (ежемесячный обязательный) |
| Потребкредит | Расход (из кредитных денег) | credit_disbursement | Перевод | Расход (рег.) | **Да** |
| Автокредит | Расход (нерег.) | credit_disbursement | Перевод | Расход (рег.) | **Да** |
| Ипотека | Расход (нерег.) | credit_disbursement | Перевод | Расход (рег.) | **Да** |

> **Примечание по кредитной карте:** при полном ежемесячном погашении КК — тело не вычитается из свободных средств, т.к. расход уже учтён в базовом потоке (покупка = расход). Если же на КК есть revolving-долг с минимальным платежом — тело минимального платежа вычитается из свободных.

## Досрочное погашение

`credit_early_repayment` — всегда перевод (тело). Не попадает в расходную аналитику, не влияет на [[Поток]]. Влияет на [[Капитал]] (долг уменьшается, кэш уменьшается -> нетто около 0) и на [[DTI]] (при полном закрытии кредита -> DTI падает).

---

## Тегирование кредитных денег (потребкредит)

Для потребительских кредитов, где деньги приходят на дебет и смешиваются с зарплатой:

**Три линии защиты от двойного учёта:**
1. Умная подсказка при импорте (совпадение суммы disbursement +/- 10%)
2. Ручная галочка «оплачено в кредит» на транзакции
3. Кредитный конверт — месячный батч-ввод: «сколько ушло на ремонт?»

**Данные:** `Account.credit_envelope_remaining`, `Account.credit_source_id`, `Account.tagged_credit_balance`.

## Рассрочки

Ежемесячный платёж по рассрочке увеличивает [[DTI]] так же, как кредитный платёж.
Три рассрочки по 5 000 = 15 000/мес обязательных платежей.
