# Страница — Review (Проверка транзакций)
#страница
> Проверка и исправление транзакций после импорта.

---

## Назначение

Страница отвечает на вопрос: **«Правильно ли система разобрала мою выписку?»**

После импорта транзакции попадают сюда с флагом `needs_review: true`. Пользователь проходит по списку: подтверждает категорию, исправляет ошибки, разделяет составные транзакции.

---

## Основные элементы

### Очередь на проверку

Список транзакций, требующих внимания. Каждая строка показывает: дату, описание, сумму, предложенную категорию, счёт. Цветовые бейджи статуса.

### Действия по каждой транзакции

- **Подтвердить** — согласиться с предложенной категорией
- **Изменить категорию** — выбрать другую через SearchSelect
- **Изменить счёт** — если система определила неверно
- **Разделить (Split)** — разбить на несколько категорий
- **Удалить** — убрать ошибочную или дублирующуюся запись

### Создание на лету

Прямо из экрана проверки можно создать:
- Новый счёт (AccountDialog)
- Новую категорию (CategoryDialog)

---

## Зачем отдельная страница

Проверка — это отдельный рабочий режим. На странице Транзакций пользователь ищет конкретные записи. На странице Проверки — последовательно обрабатывает очередь. Разные задачи — разные интерфейсы.
