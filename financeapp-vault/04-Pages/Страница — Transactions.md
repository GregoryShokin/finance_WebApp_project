# Страница — Transactions (Транзакции)
#страница
> Полный список транзакций с фильтрацией, поиском, созданием и редактированием.

---

## Назначение

Страница отвечает на вопрос: **«Что конкретно происходило с моими деньгами?»**

Это рабочая таблица — здесь пользователь ищет конкретные операции, фильтрует по дате/категории/счёту, исправляет ошибки импорта, создаёт ручные записи.

---

## Основные элементы

### Таблица транзакций (TransactionsList)

Список операций с основной информацией: дата, описание, категория, сумма, счёт. Цветовая индикация: доходы зелёным, расходы обычным текстом.

### Панель фильтров (TransactionFilters)

Сворачиваемая панель с фильтрами:
- **Поиск** — по описанию
- **Счёт** — выпадающий список
- **Категория** — выпадающий список
- **Приоритет категории** — обязательные / второстепенные / целевые / активные доходы / пассивные доходы
- **Тип** — доход / расход
- **Тип операции** — обычный, перевод, кредитный платёж, инвестиция, долг и т.д.
- **Диапазон дат** — от / до
- **Диапазон сумм** — от / до
- **Требует проверки** — фильтр для транзакций после импорта

### Форма транзакции (TransactionForm)

Универсальная форма для создания и редактирования. Адаптируется под тип операции:
- **Обычная** — счёт, категория, сумма, дата, описание
- **Перевод** — счёт-источник + счёт-получатель
- **Кредитный платёж** — счёт списания + кредит + разбивка (основной долг / проценты)
- **Долг** — контрагент + направление (дал / взял / вернул / мне вернули)
- **Инвестиция** — покупка / продажа
- **Рассрочка** — при выборе счёта типа `installment_card` появляется блок: чекбокс «Это покупка в рассрочку», срок в месяцах, ежемесячный платёж, название покупки. При создании автоматически появится запись рассрочки.

### Разделение транзакции (Split)

Возможность разбить одну транзакцию на несколько категорий. Пример: чек из магазина 5 000 ₽ → «Продукты» 3 000 ₽ + «Бытовая химия» 2 000 ₽.

---

## Кнопки и действия

- **Создать транзакцию** — открывает форму
- **Редактировать** — клик по строке
- **Удалить** — с отложенным подтверждением
- **Удалить за период** — массовое удаление с выбором дат и (опционально) счёта
- **Фильтры** — сворачиваемая панель

---

## Создание сущностей на лету

Прямо из формы транзакции можно создать:
- Новый счёт (если ввёл название, которого нет)
- Новую категорию (с автоподбором типа доход/расход)
- Нового контрагента (для долгов)

---

## Данные

- Транзакции: `GET /transactions` (с фильтрами)
- Счета, Категории, Контрагенты, Цели — для выпадающих списков в форме
