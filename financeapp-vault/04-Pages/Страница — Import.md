# Страница — Import (Импорт выписок)
#страница
> Загрузка банковских выписок и автоматическое распознавание транзакций.

---

## Назначение

Страница отвечает на вопрос: **«Как загрузить данные из банка в приложение?»**

Импорт — основной способ наполнения приложения данными. Пользователь загружает выписку (PDF, Excel, CSV), система нормализует данные, распознаёт счёт, автоматически категоризирует транзакции и выявляет дубликаты.

---

## Основные элементы

### Блок свежести данных (AccountFreshnessBlock)

Сворачиваемый блок вверху страницы: показывает, когда последний раз обновлялись данные по каждому счёту. Мотивирует загрузить свежую выписку.

### Форма загрузки

Двухколоночный макет:
- **Левая часть:** зона загрузки файла (drag & drop или кнопка)
- **Правая часть:** очередь выписок (ImportQueue)

Две кнопки:
- **«Загрузить и проверить»** — немедленная обработка
- **«Добавить в очередь»** — отложенная обработка (пакетный импорт)

### Визард импорта (ImportWizard)

Пошаговый процесс после загрузки:
1. Определение формата и банка
2. Маппинг колонок (если формат не распознан)
3. Предпросмотр транзакций
4. Проверка и подтверждение

### Очередь импорта (ImportQueue)

Список незавершённых сессий импорта. Клик по сессии возобновляет визард. Каждая сессия показывает: дату, файл, статус, количество строк.

### Автораспознавание

PDF-парсер извлекает номер договора и автоматически подбирает счёт. Показывает toast «Определён счёт: Тинькофф Дебетовая».

## Альтернативный канал: Telegram-бот

Пользователь может отправить выписку через `@financeapp_import_bot` в Telegram. Бот принимает PDF, Excel, CSV. Настройка — через одноразовый код на странице [[Страница — Settings|Настройки]].

---

## Конвейер обработки

Загруженный файл проходит через 4 этапа:
1. **Нормализация** — приведение к единому формату (даты, суммы, описания)
2. **Распознавание** — определение банка и счёта по содержимому
3. **Автокатегоризация** — подбор категории по правилам и истории
4. **Дедупликация** — выявление транзакций, которые уже есть в системе
