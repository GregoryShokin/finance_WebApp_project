# Целевой потребкредит без пометки — анализ решений

**Дата:** апрель 2026
**Проблема:** [[Стресс-тест кредитной модели#Сценарий Е]]
**Серьёзность:** ВЫСОКАЯ — самый распространённый тип кредита (авто, техника)

---

## Суть проблемы

Автокредит 500к. Деньги приходят на дебетовый счёт. Пользователь покупает машину одной транзакцией (500к с дебетовой карты). Если не поставил галочку «оплачено в кредит» — покупка считается обычным расходом → СМО = −405к, двойной счёт.

**Существующее решение:** галочка «оплачено в кредит» + автоподсказка при наличии свежего кредита (<30 дней).

**Проблема:** полностью зависит от действия пользователя. Забыл галочку — аналитика сломана.

---

## Решение: усиление через тегирование (из проблемы #2)

Механизм тегирования из [[Потребкредит на общие нужды — анализ решений]] закрывает эту проблему автоматически.

### Как это работает

1. Пользователь берёт кредит 500к → `credit_disbursement` → деньги на дебете
2. **Тегирование** помечает 500к на дебетовом счёте как кредитные
3. Пользователь импортирует выписку → покупка «Автосалон, 500к»
4. Система видит совпадение: на этом счёте 500к тегированных + покупка на ~500к
5. **Умная подсказка:** «Покупка 500к в "Автосалон" — похоже, сделана на кредитные деньги. На счёте 500к кредитных средств. Отнести к кредиту?»
6. Пользователь подтверждает → покупка исключена из СМО, тег уменьшен, конверт уменьшен

### Улучшение автоподсказки

| | Текущая подсказка | С тегированием |
|---|---|---|
| **Триггер** | Есть свежий кредит (<30 дней) | Есть тегированные деньги на этом счёте |
| **Точность** | Низкая (любая покупка триггерит) | Высокая (сумма ≈ тегированный остаток) |
| **Контекст** | «У вас есть кредит» | «На счёте 500к кредитных. Покупка 500к — совпадение?» |
| **Ложные срабатывания** | Много (каждая покупка за 30 дней) | Мало (только при совпадении суммы) |

### Три линии защиты от забывчивости

| Линия | Механизм | Когда срабатывает |
|---|---|---|
| **1. Умная подсказка** | Тегирование → совпадение суммы при импорте | Сразу при импорте |
| **2. Галочка вручную** | «Оплачено в кредит» на транзакции | При ручном вводе / редактировании |
| **3. Конверт (страховка)** | «Сколько из кредита потрачено?» в конце месяца | После импорта, раз в месяц |

Даже если пользователь проигнорировал подсказку и не поставил галочку — конверт в конце месяца позволяет исправить ситуацию одним числом.

---

## Связь с проблемами #2 и #5

Проблемы #2, #3 и #5 решаются **одним механизмом** (тегирование + конверт) на разных уровнях:

| Проблема | Тегирование решает | Конверт решает |
|---|---|---|
| **#2** Потребкредит общий (30+ транзакций) | Видимость кредитных денег | Одно число вместо 30 галочек |
| **#3** Целевой кредит (1 покупка) | Умная подсказка при импорте | Страховка если забыл |
| **#5** Ложные доступные финансы | «600к *(в т.ч. 500к кредитных)*» | — |

---

## Критерии умной подсказки

Система предлагает «отнести к кредиту» если:

1. На счёте есть тегированные кредитные деньги (`tagged_credit_balance > 0`)
2. Появилась покупка, сумма которой:
   - Равна или близка к тегированному остатку (±10%), **ИЛИ**
   - Превышает 50% тегированного остатка (крупная покупка)
3. Покупка сделана с того же счёта, где лежат тегированные деньги

**Не предлагает** для мелких покупок (продукты, кафе) — они очевидно не связаны с целевым кредитом.
