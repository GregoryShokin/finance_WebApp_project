# Текущее лучшее решение

> **Статус: АРХИВ (апрель 2026).** Заменено мультиметрической системой. Текущая архитектура описана в [[Поток]], [[Капитал]], [[DTI]], [[Запас]].

---

## Что произошло

Этот документ описывал подход «Покупка на кредит = не расход, расход при погашении» с единым фундаментом СМО. При переходе на мультиметрическую систему (апрель 2026) весь подход пересмотрен:

| Было (единый СМО) | Стало (мультиметрика) |
|---|---|
| СМО — единственный фундамент | Четыре метрики: Поток, Капитал, DTI, Запас |
| Покупка на кредитку ≠ расход | Поток = реальное движение кэша по счетам |
| Погашение = расход (сектор «Кредитные платежи») | Полный платёж (тело + %) всегда в базовом потоке |
| Два слоя аналитики (категорийная + расходная) | Единый поток: базовый (регулярные) + полный (всё) |
| 5 типов классификации поступлений | Три режима на категории: регулярная / нерегулярная / проект |
| FIFO-очередь для рассрочки | Не нужна — только ежемесячные платежи на счёте |
| Тегирование потребкредита | Disbursement = нерегулярный доход, расходы = нерегулярные |

## Главный принцип (обновлённый)

> **Поток = реальное движение кэша по банковским счетам пользователя.**

Никаких виртуальных транзакций. Если операция не прошла через счёт пользователя — её нет в Потоке.

## Как работают типы кредитов

| Тип кредита | Что на счёте | В базовом потоке | В полном потоке | Капитал |
|---|---|---|---|---|
| **Рассрочка** | Только ежемес. платёж | Платёж | Платёж | Долг ↓ на тело |
| **Потребкредит** (на счёт) | Disbursement + расходы + платёж | Платёж | Всё | Долг ↓ на тело |
| **Автокредит / ипотека** | Только ежемес. платёж | Платёж | Платёж | Долг ↓ на тело, актив ± |
| **Кредитная карта** | Расходы + погашение | Регулярные расходы | Всё | Долг меняется |

## Что решает мультиметрика (что ломалось в старой модели)

- **Кешбэк-пользователь** — все расходы через кредитку: теперь расходы видны в категориях, а не в «Кредитных платежах»
- **Двойной счёт потребкредита** — disbursement + расходы взаимогасятся в полном потоке (оба нерегулярные)
- **Импульсная покупка на кредитку** — видна в полном потоке и Капитале
- **Крупные покупки ломают среднее** — нерегулярный флаг → не в базовом потоке
- **Досрочное погашение** — виртуальный расход не нужен; досрочка видна в полном потоке и улучшает Капитал

## Нерешённые вопросы (из прежней модели)

Большинство нерешённых вопросов снялись при переходе на мультиметрику:
- ~~Кешбэк-пользователь~~ → решено (расходы = расходы в категориях)
- ~~Потребкредит двойной счёт~~ → решено (disbursement нерегулярный, расходы нерегулярные)
- ~~Досрочка проваливает СМО~~ → решено (Поток + Капитал)
- ~~DTI при минимальном платеже~~ → остаётся; см. [[Долговая спираль — анализ решений]]
- ~~Определение денежного потока~~ → решено (полный поток = Δ кэша)
