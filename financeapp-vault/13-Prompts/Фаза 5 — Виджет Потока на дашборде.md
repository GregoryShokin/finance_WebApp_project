#### 3.3 SVG донат

Размеры: viewBox="0 0 240 240", ширина 240px, высота 240px.

Два кольца:
- Внешнее: cx=120, cy=120, r=80, stroke-width=14 (Доходы)
- Внутреннее: cx=120, cy=120, r=58, stroke-width=12 (Расходы + кредиты)

Максимальная дуга = 75% окружности:
- outer_max = 2π × 80 × 0.75 = 376.99
- inner_max = 2π × 58 × 0.75 = 273.32
- outer_circumference = 502.65
- inner_circumference = 364.42

Поворот обоих колец: `transform="rotate(135 120 120)"` — начало дуги в левом нижнем углу.

Фоновые треки (светлая тема): `stroke="#e2e8f0"` (slate-200)

Сегменты — последовательные дуги одного кольца. Каждый сегмент:
```
stroke-dasharray: `${segLen} ${circumference}`
stroke-dashoffset: -accumulatedOffset   (отрицательный = смещение вперёд)
```

Цвета сегментов (по спецификации, не менять):
- Доходы: `#06b6d4` (cyan)
- Повседневные расходы: `#f43f5e` (rose)
- Тело кредитов: `#94a3b8` (slate)
- Инвестиции: `#8b5cf6` (violet)
- Компенсатор (Δ долга КК): `#fbbf24` (amber)
- Disbursement: `#60a5fa` (blue)

Центр доната — текст в SVG:
```tsx
<text x="120" y="118" textAnchor="middle" className="text-xl font-bold fill-slate-900">
  {formatCenterValue(centerAmount)}
</text>
<text x="120" y="136" textAnchor="middle" className="text-[10px] fill-slate-400 uppercase tracking-widest">
  {centerLabel}
</text>
```

Зафиксированное число: знак + число + «₽». Если отрицательное — `fill-rose-600`.

#### 3.4 Что в кольцах для каждого таба

Нужно рассчитать суммы для каждого сегмента, чтобы нарисовать дуги. Источник данных — `/metrics/summary`. Для детальных сумм (регулярные расходы, нерегулярные и т.д.) используй транзакции месяца (вычисляются в `computeFlowBreakdown` — новая функция в `dashboard-data.ts`, шаг 4).

Пока `/metrics/summary` не возвращает детальную разбивку (регулярные отдельно от нерегулярных), используй следующий upрощённый маппинг:

**Базовый таб:**
- Outer: `[{color: cyan, value: regular_income}]`
- Inner: `[{color: rose, value: regular_expenses}]`
- max = max(regular_income, regular_expenses)
- Центр: `basic_flow`, лейбл: «БАЗОВЫЙ»

Для regular_income и regular_expenses: вычисли из транзакций за текущий месяц (аналогично тому, как сейчас делает `computeFlow` в `dashboard-data.ts`, но с фильтром `is_regular`).

**Свободный таб:**
- Outer: `[{color: cyan, value: regular_income}]`
- Inner: `[{color: rose, value: regular_expenses}, {color: slate, value: credit_body_payments}]`
- max = max(regular_income, regular_expenses + credit_body_payments)
- Центр: `free_capital`, лейбл: «СВОБОДНЫЙ»

`credit_body_payments` = `summary.flow.credit_body_payments`

**Полный таб:**
- Outer сегменты: `[{cyan, all_income}, {amber, cc_compensator > 0 ? cc_compensator : 0}]`
- Inner сегменты: `[{rose, all_expenses}, {slate, credit_body}, {violet, investments > 0 ? investments : 0}]`
- Если `cc_compensator < 0` (долг уменьшился) — добавляем amber во внутреннее кольцо
- max = max(sum_outer, sum_inner)
- Центр: `full_flow`, лейбл: «Δ КЭША»

`all_income`, `all_expenses` — все транзакции месяца (не только регулярные). `investments` = `investmentBuyAmount` prop.

### Шаг 4. Добавить `computeFlowBreakdown` в dashboard-data.ts

Нужна функция для вычисления сумм, которые доступны только из транзакций (детальные разбивки по regular/irregular, all_income, all_expenses, investments).

```typescript
export interface FlowBreakdown {
  regularIncome: number;
  regularExpenses: number;
  allIncome: number;
  allExpenses: number;
  investmentBuy: number;   // сумма investment_buy за месяц
}

export function computeFlowBreakdown(transactions: Transaction[]): FlowBreakdown {
  const now = new Date();
  const key = monthKey(now);
  let regularIncome = 0, regularExpenses = 0;
  let allIncome = 0, allExpenses = 0;
  let investmentBuy = 0;

  for (const tx of transactions) {
    if (!tx.affects_analytics) continue;
    if (txMonthKey(tx) !== key) continue;
    const amount = toNum(tx.amount);

    if (tx.type === 'income' && tx.operation_type !== 'credit_disbursement') {
      allIncome += amount;
      if (tx.is_regular) regularIncome += amount;
    }
    if (tx.type === 'expense' && tx.operation_type !== 'transfer' && tx.operation_type !== 'credit_early_repayment') {
      allExpenses += amount;
      if (tx.is_regular) regularExpenses += amount;
    }
    if (tx.operation_type === 'investment_buy') {
      investmentBuy += amount;
    }
  }

  return { regularIncome, regularExpenses, allIncome, allExpenses, investmentBuy };
}
```

## Подводные камни

**1. Сегментированные SVG дуги.**

Самая сложная часть. Дуги нарисованы как несколько `<circle>` с разными `stroke-dashoffset`. Ключевой момент: каждый следующий сегмент смещается на суммарную длину предыдущих:

```typescript
let offset = 0;
segments.forEach(seg => {
  const len = (seg.value / maxVal) * maxArc;
  // stroke-dasharray: `${len} ${circumference}`
  // stroke-dashoffset: -offset
  offset += len;
});
```

Отрицательный `dashoffset` — SVG интерпретирует как смещение «вперёд» по направлению дуги.

**2. `free_capital` может прийти из API как 0 или отрицательный.**

В этом случае Свободный таб показывает красное число — это правильно. Полоска инвестиций скрывается если `free_capital <= 0`.

**3. Синхронизация данных из двух источников (API + транзакции).**

FlowWidget получает данные из `/metrics/summary` (бэкенд) и `flowBreakdown` (фронтальный расчёт из транзакций). Эти числа могут немного расходиться в граничных случаях (часовые пояса, неполный месяц). Для UI это нормально — главные числа (basic_flow, free_capital, full_flow) берутся из API; детальная разбивка (regularIncome, regularExpenses) — из транзакций.

**4. Кольца «пересекаются» если inner > outer.**

При дефиците (inner длиннее outer) max = inner value. Outer дуга короче = визуально виден дефицит. Это корректное поведение (описано в UX-гипотезе § 3.4).

**5. Адаптация цветов под светлую тему.**

Мокап `trilayer-flow-widget.html` использует тёмный фон `#0f172a`. Дашборд-new — белый фон. Адаптировать:
- Фоновые кольца (track): `#e2e8f0` (slate-200) вместо `#334155`
- Текст центра доната: `text-slate-900` вместо `fill-#f1f5f9`
- Заголовок карточки: `text-slate-900` вместо `text-#f1f5f9`
- Подсказка: цветные блоки с `bg-*-50` вместо `rgba(..., 0.08)`

Если `dashboard-new/page.tsx` и `section-analytics.tsx` оба импортируют `MetricsSummary` — убедись, что тип доступен из единого источника (`@/lib/api/metrics`).
