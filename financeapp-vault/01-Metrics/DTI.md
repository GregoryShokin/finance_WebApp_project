# DTI

#показатель #мультиметрика

> **Актуально на:** 2026-04-16 | Источник: -> [[Спецификация — Целевое состояние системы]]

> Кредитная нагрузка — какую долю **регулярного** дохода забирают обязательные платежи по кредитам. В UI везде называется **«Кредитная нагрузка»**, не DTI.

---

## Формула

```
DTI = Сумма ежемесячных платежей по всем кредитам (тело + %) / средний регулярный доход x 100%
```

**Числитель:** полный ежемесячный платёж (тело + проценты) по всем кредитным счетам, включая рассрочки.

Источник платежа:
1. Последний фактический платёж из транзакций
2. Fallback: поле `Account.monthly_payment`

**Знаменатель:** ТОЛЬКО регулярный доход (средний за 3 мес., категории с флагом `regular`). Нерегулярные поступления (бонусы, продажа активов) исключены, чтобы не занижать DTI искусственно.

> [!note] Тело и проценты
> Хотя в новой модели тело = `transfer`, а проценты = `expense`, для DTI используется **полный** платёж (тело + проценты), т.к. DTI отвечает на вопрос «сколько дохода уходит банку», а банку уходит всё.

## Зоны

| Зона | Значение | Значение для пользователя |
|------|----------|--------------------------|
| Норма | < 30% | Кредиты под контролем |
| Допустимо | 30-40% | «2 из 5 дней — на банк» |
| Опасно | > 40% | Высокий риск при потере дохода |
| Критично | > 60% | Предбанкротная зона |

## Взаимодействие с другими метриками

DTI показывает кредитную нагрузку отдельно от [[Поток|Потока]]. Пользователь с базовым потоком +60к и DTI 40% понимает: «60к — мой операционный плюс, но значительная часть дохода уходит банку».

Связь с Потоком: тело кредитных платежей вычитается из базового потока в метрике «Свободные средства» (см. [[Поток]]).

## Где используется

- [[Порог финансовой устойчивости]] — условие DTI < 40%
